Examples 6.1 — Web Components

BASIC (1–13)

1. customElements.define — registering a custom element

class MyGreeting extends HTMLElement {}

customElements.define('my-greeting', MyGreeting);
// First arg: tag name (must contain a hyphen)
// Second arg: class extending HTMLElement

// Use it in HTML:
// <my-greeting></my-greeting>

customElements.define(tagName, constructor) registers a new autonomous custom element globally.

2. Extending HTMLElement — the base class

class UserCard extends HTMLElement {
  // Custom elements MUST extend HTMLElement (or a subclass).
  // The class gets all native element methods: getAttribute,
  // setAttribute, addEventListener, remove(), etc.
}

customElements.define('user-card', UserCard);

// ✅ Valid — extends HTMLElement
// ❌ Invalid — plain class without HTMLElement base

Every autonomous custom element class must extend HTMLElement to integrate with the browser's element upgrade system.

3. connectedCallback — element added to the DOM

class MyBanner extends HTMLElement {
  connectedCallback() {
    // Runs every time the element is inserted into a document.
    // Safe to access this.ownerDocument, parent, attributes here.
    console.log('my-banner connected to DOM');
    this.textContent = 'Hello from web component!';
  }
}

customElements.define('my-banner', MyBanner);

connectedCallback fires after the element is attached to the document — the right place to set up UI and start work.

4. Adding innerHTML inside connectedCallback

class AlertBox extends HTMLElement {
  connectedCallback() {
    const type = this.getAttribute('type') || 'info';
    this.innerHTML = `
      <div class="alert alert--${type}">
        <strong>${type.toUpperCase()}</strong>
        <slot></slot>
      </div>
    `;
  }
}

customElements.define('alert-box', AlertBox);

// Usage: <alert-box type="error">Something went wrong</alert-box>

Setting innerHTML in connectedCallback renders the component's template. Use getAttribute to read configuration at connect time.

5. Using a custom element in HTML

<!-- Register the element in a script first -->
<script>
  class MyTag extends HTMLElement {
    connectedCallback() { this.textContent = 'I am custom!'; }
  }
  customElements.define('my-tag', MyTag);
</script>

<!-- Then use it anywhere in the document -->
<my-tag></my-tag>

<!-- Or use the tag before define() — the browser upgrades it later -->
<my-tag></my-tag>
<script>customElements.define('my-tag', MyTag);</script>

Custom elements can appear before or after their define() call. Unresolved elements are upgraded once define() runs.

6. Custom element names must contain a hyphen

// ✅ Valid names — contain a hyphen
customElements.define('my-component', MyComponent);
customElements.define('x-button', XButton);
customElements.define('app-header', AppHeader);
customElements.define('ui-card--featured', Card);

// ❌ Invalid — no hyphen (reserved for built-in elements)
// customElements.define('mycomponent', ...);  // SyntaxError
// customElements.define('button', ...);        // SyntaxError
// customElements.define('h1', ...);            // SyntaxError

The hyphen requirement prevents name collisions with current and future HTML elements, which never contain hyphens.

7. Class fields vs attributes — initial state

class ThemeButton extends HTMLElement {
  // Class field — JS-only internal state (not visible in HTML)
  #clickCount = 0;

  // Attribute — reflected in HTML, accessible to CSS selectors
  // <theme-button color="blue">

  connectedCallback() {
    this.addEventListener('click', () => {
      this.#clickCount++;
      // Read attribute at runtime:
      const color = this.getAttribute('color') ?? 'gray';
      console.log(`Clicked ${this.#clickCount}x, color: ${color}`);
    });
  }
}

customElements.define('theme-button', ThemeButton);

Use private class fields for internal JS state and HTML attributes for data that should be serialisable, CSS-selectable, or HTML-authorable.

8. Autonomous vs customized built-in elements

// ── Autonomous custom element ─────────────────────────────────────
// Extends HTMLElement directly; used as its own tag.
class MyCard extends HTMLElement {}
customElements.define('my-card', MyCard);
// <my-card></my-card>

// ── Customized built-in element ───────────────────────────────────
// Extends a specific built-in (e.g. HTMLButtonElement).
class FancyButton extends HTMLButtonElement {
  connectedCallback() { this.style.background = 'hotpink'; }
}
customElements.define('fancy-button', FancyButton, { extends: 'button' });
// <button is="fancy-button">Click me</button>

// Note: Safari does NOT support customized built-ins without a polyfill.

Autonomous elements are brand-new tags; customized built-ins extend existing HTML elements via the { extends } third argument and the is= attribute.

9. customElements.get() — retrieve a registered constructor

class MyWidget extends HTMLElement {}
customElements.define('my-widget', MyWidget);

// Get the constructor back by tag name:
const Ctor = customElements.get('my-widget');
console.log(Ctor === MyWidget); // true

// Returns undefined for unregistered names:
const missing = customElements.get('not-defined');
console.log(missing); // undefined

// Useful for conditional registration:
if (!customElements.get('my-widget')) {
  customElements.define('my-widget', MyWidget);
}

customElements.get() lets you check whether a name is already registered and retrieve the constructor — essential for avoiding duplicate define() errors.

10. customElements.whenDefined() — promise that resolves on registration

// whenDefined returns a Promise that resolves with the constructor
// as soon as the element is defined (or immediately if already defined).

customElements.whenDefined('lazy-chart').then((ChartClass) => {
  console.log('lazy-chart is now defined:', ChartClass);
  document.querySelectorAll('lazy-chart').forEach(el => el.refresh());
});

// Useful with dynamic import:
async function loadChart() {
  await import('./chart-component.js');
  await customElements.whenDefined('lazy-chart');
  console.log('Safe to interact with lazy-chart elements');
}

whenDefined() prevents race conditions when a custom element's script loads asynchronously — await it before querying or upgrading elements.

11. dataset.initialized pattern — guard against double-init

class SafeInit extends HTMLElement {
  connectedCallback() {
    // Guard: connectedCallback can fire multiple times if the element
    // is moved between documents or removed/re-added to the DOM.
    if (this.dataset.initialized) return;
    this.dataset.initialized = 'true';

    this.innerHTML = '<p>Initialized once!</p>';
    this.addEventListener('click', this.#handleClick);
  }

  #handleClick = () => console.log('clicked');
}

customElements.define('safe-init', SafeInit);

connectedCallback fires each time the element re-enters the DOM. Set a flag (dataset, WeakMap, or private field) to skip repeated initialisation.

12. Passing data via attributes

class UserAvatar extends HTMLElement {
  connectedCallback() {
    const name = this.getAttribute('name') ?? 'Anonymous';
    const src  = this.getAttribute('src')  ?? 'default.png';
    const size = this.getAttribute('size') ?? '40';

    this.innerHTML = `
      <img
        src="${src}"
        alt="${name}'s avatar"
        width="${size}"
        height="${size}"
        style="border-radius:50%"
      >
    `;
  }
}

customElements.define('user-avatar', UserAvatar);

// <user-avatar name="Alice" src="alice.jpg" size="64"></user-avatar>

HTML attributes are the standard way to pass initial configuration — they're HTML-author friendly, serialisable, and work with SSR.

13. constructor() basics — what belongs there

class MyEl extends HTMLElement {
  constructor() {
    super(); // MUST be first — initialises the HTMLElement base

    // ✅ OK in constructor:
    this.#state = { count: 0 };     // set up private state
    this.attachShadow({ mode: 'open' }); // attach shadow root

    // ❌ NOT safe in constructor (element not yet in DOM):
    // this.getAttribute('x')  — attributes may not be stamped yet
    // this.innerHTML = '...'  — shadow root exists but upgrade still running
    // this.parentNode          — null; not connected yet
  }

  #state;

  connectedCallback() {
    // Do DOM work here instead
    this.shadowRoot.innerHTML = '<p>Ready</p>';
  }
}

customElements.define('my-el', MyEl);

The constructor should only call super(), attach shadow, and set up private fields. All DOM manipulation and attribute reads belong in connectedCallback.

INTERMEDIATE (14–26)

14. disconnectedCallback — cleanup when removed from DOM

class LiveClock extends HTMLElement {
  #intervalId = null;

  connectedCallback() {
    this.#render();
    // Start a repeating timer when added to DOM
    this.#intervalId = setInterval(() => this.#render(), 1000);
  }

  disconnectedCallback() {
    // ✅ Clean up to prevent memory leaks
    clearInterval(this.#intervalId);
    this.#intervalId = null;
    console.log('Clock stopped — element removed from DOM');
  }

  #render() {
    this.textContent = new Date().toLocaleTimeString();
  }
}

customElements.define('live-clock', LiveClock);

disconnectedCallback fires whenever the element leaves the document. Use it to clear timers, abort fetches, and remove external event listeners.

15. attributeChangedCallback — react to attribute changes

class ColorSwatch extends HTMLElement {
  static observedAttributes = ['color'];

  attributeChangedCallback(name, oldValue, newValue) {
    // name     — which attribute changed
    // oldValue — previous value (null if first set)
    // newValue — new value (null if removed)
    if (name === 'color') {
      this.style.backgroundColor = newValue ?? 'transparent';
      console.log(`color: ${oldValue} → ${newValue}`);
    }
  }
}

customElements.define('color-swatch', ColorSwatch);

// In JS:
// const el = document.querySelector('color-swatch');
// el.setAttribute('color', 'tomato');  // triggers callback

attributeChangedCallback receives the attribute name and both old and new values. It only fires for attributes listed in observedAttributes.

16. observedAttributes static getter — declaring watched attributes

class ProgressBar extends HTMLElement {
  // Must be a static getter (or static class field) returning an array.
  // Only attributes in this list trigger attributeChangedCallback.
  static get observedAttributes() {
    return ['value', 'max', 'label'];
  }

  // Equivalent modern syntax (class field):
  // static observedAttributes = ['value', 'max', 'label'];

  attributeChangedCallback(name, _old, newVal) {
    this.#render();
  }

  #render() {
    const val = Number(this.getAttribute('value') ?? 0);
    const max = Number(this.getAttribute('max')   ?? 100);
    const pct = Math.round((val / max) * 100);
    if (this.shadowRoot)
      this.shadowRoot.querySelector('.bar').style.width = pct + '%';
  }
}

customElements.define('progress-bar', ProgressBar);

observedAttributes is a static property that acts as an allowlist — unlisted attributes never trigger the callback, which keeps the component efficient.

17. Reflecting attributes to properties — two-way sync

class RatingEl extends HTMLElement {
  static observedAttributes = ['value'];

  // Property getter reads from attribute (source of truth in HTML)
  get value() {
    return Number(this.getAttribute('value') ?? 0);
  }

  // Property setter writes to attribute so CSS and HTML stay in sync
  set value(v) {
    this.setAttribute('value', String(v));
  }

  attributeChangedCallback() {
    this.#update();
  }

  #update() {
    this.textContent = '★'.repeat(this.value);
  }
}

customElements.define('rating-el', RatingEl);

// Both work identically:
// el.setAttribute('value', '4');
// el.value = 4;

Reflecting attributes to properties keeps JS and HTML in sync: the attribute is the single source of truth, and the property is a typed JavaScript alias for it.

18. Shadow DOM with attachShadow

class FancyCard extends HTMLElement {
  constructor() {
    super();
    // attachShadow creates an encapsulated subtree.
    // Returns the ShadowRoot — also stored as this.shadowRoot.
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.innerHTML = `
      <style>
        :host { display: block; border: 2px solid navy; padding: 12px; }
        h2    { color: navy; margin: 0 0 8px; }
      </style>
      <h2>Card Title</h2>
      <slot></slot>
    `;
  }
}

customElements.define('fancy-card', FancyCard);

// <fancy-card>Card body content here.</fancy-card>

attachShadow() creates an encapsulated DOM subtree. Styles inside the shadow root do not leak out, and outside styles do not leak in.

19. Shadow DOM mode — open vs closed

// ── open mode ────────────────────────────────────────────────────
class OpenEl extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
    // this.shadowRoot is the ShadowRoot
  }
}
// From outside: el.shadowRoot.querySelector('p')  ← works

// ── closed mode ──────────────────────────────────────────────────
class ClosedEl extends HTMLElement {
  #root; // store reference internally
  constructor() {
    super();
    this.#root = this.attachShadow({ mode: 'closed' });
    // this.shadowRoot === null  ← no external access
  }
}
// From outside: el.shadowRoot  → null  ← intentionally blocked

// closed mode is NOT a security boundary — devtools can still access it.
// Use closed to signal "implementation detail, do not poke".

Open shadow roots expose shadowRoot externally; closed roots return null. Neither is a security boundary — closed is an API encapsulation hint.

20. Slots in Shadow DOM — projecting light DOM children

class MyPanel extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <div class="panel">
        <!-- Default slot: receives all un-named children -->
        <slot>(no content provided)</slot>
      </div>
    `;
  }
}
customElements.define('my-panel', MyPanel);

// Usage:
// <my-panel>
//   <p>This paragraph is projected into the slot.</p>
// </my-panel>

// The <p> stays in the light DOM but is *rendered* inside the shadow.

A <slot> in the shadow DOM projects light DOM children into the rendered position without moving them in the DOM tree.

21. Named slots — routing children to specific positions

class ArticleLayout extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <article>
        <header><slot name="heading"></slot></header>
        <section><slot></slot></section>
        <footer><slot name="footer">© 2026</slot></footer>
      </article>
    `;
  }
}
customElements.define('article-layout', ArticleLayout);

// Usage:
// <article-layout>
//   <h1 slot="heading">My Post</h1>
//   <p>Body text goes to the default slot.</p>
//   <p slot="footer">Posted 2026-01-01</p>
// </article-layout>

Elements with slot="name" are projected into the matching <slot name="name">. Unslotted children go to the default (unnamed) slot.

22. CSS in Shadow DOM with :host

class PillBadge extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <style>
        /* :host refers to the custom element itself */

          background: #e0e7ff;
          color: #3730a3;
          font-size: 12px;
          font-weight: 600;
        }
        /* :host([variant="danger"]) — conditional on attribute */

:host selects the shadow host element from inside the shadow DOM. :host([attr]) applies styles conditionally based on host attributes.

23. CSS custom properties crossing the shadow boundary

<!-- component definition -->
class ThemedBox extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <style>
        :host {
          display: block;
          /* CSS custom properties DO cross the shadow boundary */
          background: var(--box-bg, #f0f9ff);
          color:      var(--box-color, #0c4a6e);
          padding:    var(--box-padding, 16px);
          border-radius: 8px;
        }
      </style>
      <slot></slot>
    `;
  }
}
customElements.define('themed-box', ThemedBox);

// From outside the component (light DOM CSS):
// themed-box { --box-bg: #fdf4ff; --box-color: #581c87; }

// This is the primary theming API for shadow DOM components.

Regular CSS properties are blocked by shadow DOM, but CSS custom properties (--*) inherit through shadow boundaries, making them the standard theming API.

24. adoptedCallback — element moved to a new document

class TrackingEl extends HTMLElement {
  connectedCallback()    { console.log('connected to', this.ownerDocument.title); }
  disconnectedCallback() { console.log('disconnected'); }

  adoptedCallback(oldDoc, newDoc) {
    // Fires when the element is moved via document.adoptNode()

    console.log('adopted from', oldDoc.title, 'to', newDoc.title);
  }
}
customElements.define('tracking-el', TrackingEl);

// Example: move element to an iframe's document
// const iframe = document.querySelector('iframe');
// const el     = document.querySelector('tracking-el');
// iframe.contentDocument.adoptNode(el); // triggers adoptedCallback

adoptedCallback fires when a custom element is moved between documents (e.g. into an iframe). It is rarely needed but useful for cross-document widgets.

25. Form-associated custom elements

class StarRating extends HTMLElement {
  // Opt-in to form association
  static formAssociated = true;

  #internals;
  #value = '';

  constructor() {
    super();
    // ElementInternals gives access to form APIs
    this.#internals = this.attachInternals();
    this.attachShadow({ mode: 'open' });
  }

  // Form APIs now available:
  get form()     { return this.#internals.form; }
  get name()     { return this.getAttribute('name'); }
  get validity() { return this.#internals.validity; }

  #setValue(v) {
    this.#value = v;
    // Notify the form of the new value
    this.#internals.setFormValue(v);
  }
}

customElements.define('star-rating', StarRating);

// <form>
//   <star-rating name="rating"></star-rating>
//   <button type="submit">Submit</button>
// </form>

Setting formAssociated = true and calling attachInternals() lets a custom element participate in forms: it submits values, validates, and resets like native inputs.

26. CSS ::part() — styling shadow DOM from outside

class MyButton extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <style>
        button { padding: 8px 16px; border: none; border-radius: 4px; }
      </style>
      <!-- 'part' attribute exposes the element to outside CSS -->
      <button part="button">
        <span part="label"><slot></slot></span>

// my-button::part(button) { background: royalblue; color: white; }
// my-button::part(label)  { font-weight: bold; letter-spacing: 1px; }

::part() lets component authors explicitly expose shadow DOM internals to outside CSS, providing a controlled styling API without breaking encapsulation.

28. Icon component

class SvgIcon extends HTMLElement {
  static observedAttributes = ['name', 'size', 'color'];

  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <style>
        :host { display:inline-block; line-height:0; }
        svg   { fill:currentColor; }
      </style>
      <svg viewBox="0 0 24 24" aria-hidden="true"></svg>
    `;
  }

  connectedCallback() { this.#update(); }
  attributeChangedCallback() { this.#update(); }

  #paths = {
    check: 'M20 6L9 17l-5-5',
    close: 'M18 6L6 18M6 6l12 12',
    star:  'M12 2l3.09 6.26L22 9.27l-5 4.87 1.18 6.88L12 17.77l-6.18 3.25L7 14.14 2 9.27l6.91-1.01L12 2z',
  };

  #update() {
    const size  = this.getAttribute('size')  ?? '24';
    const color = this.getAttribute('color') ?? 'currentColor';
    const path  = this.#paths[this.getAttribute('name')] ?? '';
    const svg   = this.shadowRoot.querySelector('svg');
    svg.setAttribute('width', size);
    svg.setAttribute('height', size);
    svg.style.color = color;
    svg.innerHTML = `<path d="${path}" stroke-width="2" fill="none" stroke="currentColor" stroke-linecap="round"/>`;
  }
}
customElements.define('svg-icon', SvgIcon);

An icon component uses observedAttributes to re-render when name/size/color change, producing inline SVG without external sprite files.

30. Star rating widget

class StarWidget extends HTMLElement {
  static observedAttributes = ['value', 'max'];
  static formAssociated = true;
  #internals;

  constructor() {
    super();
    this.#internals = this.attachInternals();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() { this.#render(); }
  attributeChangedCallback() { this.#render(); }

  #render() {
    const max = Number(this.getAttribute('max') ?? 5);
    const val = Number(this.getAttribute('value') ?? 0);
    const stars = Array.from({ length: max }, (_, i) => {
      const filled = i < val ? 'filled' : '';
      return `<button part="star" class="${filled}" data-v="${i + 1}">★</button>`;
    }).join('');
    this.shadowRoot.innerHTML = `
      <style>
        :host { display:inline-flex; gap:4px; }
        button { background:none; border:none; cursor:pointer; font-size:24px;
                 color:#ddd; padding:0; line-height:1; }
        .filled { color:#f59e0b; }
      </style>
      ${stars}
    `;
    this.shadowRoot.addEventListener('click', e => {
      const v = e.target.dataset.v;
      if (v) { this.setAttribute('value', v); this.#internals.setFormValue(v); }
    });
  }
}
customElements.define('star-widget', StarWidget);

A form-associated star rating widget re-renders on attribute change, sets the form value via ElementInternals, and exposes stars via ::part() for styling.

31. Progress component

class RingProgress extends HTMLElement {
  static observedAttributes = ['value', 'max', 'label'];

  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() { this.#render(); }
  attributeChangedCallback() { this.#render(); }

  #render() {
    const val  = Number(this.getAttribute('value') ?? 0);
    const max  = Number(this.getAttribute('max')   ?? 100);
    const lbl  = this.getAttribute('label') ?? '';
    const pct  = Math.min(Math.max(val / max, 0), 1);
    const r = 40, circ = 2 * Math.PI * r;
    const dash = (1 - pct) * circ;
    this.shadowRoot.innerHTML = `
      <style>
        :host { display:inline-block; width:100px; height:100px; }
        svg { transform:rotate(-90deg); }
        circle.track { fill:none; stroke:#e5e7eb; stroke-width:8; }
        circle.bar   { fill:none; stroke:var(--progress-color,#6366f1);
                       stroke-width:8; stroke-linecap:round;
                       transition:stroke-dashoffset .4s; }
        text { transform:rotate(90deg) translate(50%,-50%);
               transform-box:fill-box; font-size:14px; text-anchor:middle;
               dominant-baseline:middle; fill:#374151; }
      </style>
      <svg viewBox="0 0 100 100">
        <circle class="track" cx="50" cy="50" r="${r}"/>
        <circle class="bar" cx="50" cy="50" r="${r}"
          stroke-dasharray="${circ}" stroke-dashoffset="${dash}"/>
        <text x="50" y="50">${Math.round(pct*100)}%</text>
      </svg>
    `;

34. Date display component

class DateDisplay extends HTMLElement {
  static observedAttributes = ['datetime', 'format'];

  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML =
      `<time></time>`;
  }

  connectedCallback()    { this.#render(); }
  attributeChangedCallback() { this.#render(); }

  #render() {
    const dt  = this.getAttribute('datetime');
    const fmt = this.getAttribute('format') ?? 'long';
    if (!dt) return;

    const date = new Date(dt);
    const opts = {
      short:    { month:'short', day:'numeric', year:'numeric' },
      long:     { weekday:'long', year:'numeric', month:'long', day:'numeric' },
      relative: null,
    };

    const el = this.shadowRoot.querySelector('time');
    el.setAttribute('datetime', dt);
    el.textContent = fmt === 'relative'
      ? this.#relative(date)
      : date.toLocaleDateString(undefined, opts[fmt] ?? opts.long);
  }

  #relative(date) {
    const diff = Math.round((date - Date.now()) / 86400000);
    const rtf  = new Intl.RelativeTimeFormat('en', { numeric: 'auto' });
    return rtf.format(diff, 'day');
  }
}
customElements.define('date-display', DateDisplay);

// <date-display datetime="2026-06-15" format="relative"></date-display>

A date component uses Intl.RelativeTimeFormat and Intl-aware toLocaleDateString to render human-readable dates in different formats from a datetime attribute.

48. State machine pattern in a custom element

class FetchButton extends HTMLElement {
  #state = 'idle'; // 'idle' | 'loading' | 'success' | 'error'

  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() {
    this.#render();
    this.shadowRoot.addEventListener('click', () => this.#fetch());
  }

  async #fetch() {
    this.#transition('loading');
    try {
      const res  = await fetch(this.getAttribute('url') ?? '/api/data');
      const data = await res.json();
      this.dispatchEvent(new CustomEvent('data', { detail: data, bubbles: true }));
      this.#transition('success');
    } catch {
      this.#transition('error');
    }
  }

  #transition(next) {
    this.#state = next;
    this.setAttribute('state', next); // CSS hooks via [state=...]
    this.#render();
  }

  #render() {
    const labels = { idle:'Load', loading:'Loading…', success:'Done ✓', error:'Retry' };
    this.shadowRoot.innerHTML = `
      <button ?disabled="${this.#state === 'loading'}">
        ${labels[this.#state]}
      </button>
    `;
  }
}
customElements.define('fetch-button', FetchButton);

Modelling component state as an explicit state machine prevents impossible states, makes transitions traceable, and exposes the current state as an attribute for CSS hooks.

49. Intersecting custom element with IntersectionObserver

class LazySection extends HTMLElement {
  #observer = null;

  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML =
      `<style>:host{display:block;min-height:200px;}</style><slot></slot>`;
  }

  connectedCallback() {
    this.#observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          this.removeAttribute('hidden-content');
          this.setAttribute('visible', '');
          this.dispatchEvent(new Event('visible', { bubbles: true }));
          this.#observer.disconnect();
        }
      },
      { rootMargin: '0px 0px -100px 0px', threshold: 0 }
    );
    this.#observer.observe(this);
  }

  disconnectedCallback() {
    this.#observer?.disconnect();
    this.#observer = null;
  }
}
customElements.define('lazy-section', LazySection);

Connecting an IntersectionObserver in connectedCallback and disconnecting in disconnectedCallback gives elements self-managed viewport detection with zero memory leaks.

50. Custom element design API checklist

// ── Checklist for a production-ready custom element ───────────────

// 1. Name: contains a hyphen; descriptive and scoped (e.g. app-button)
// 2. Constructor: only super() + attachShadow + private fields
// 3. Attributes: list ALL in observedAttributes; reflect to properties
// 4. CSS: use :host for self-styling; expose --custom-props for theming
// 5. Slots: use named slots for structure; document the slot API
// 6. Events: dispatch CustomEvents with composed:true when needed
// 7. Cleanup: clear timers/observers/listeners in disconnectedCallback
// 8. Forms: use formAssociated + ElementInternals when needed
// 9. A11y: set role/aria-* on host or shadow; manage focus
// 10. SSR: support Declarative Shadow DOM for server-rendered content
// 11. TypeScript: extend HTMLElementTagNameMap for consumer types
// 12. Tests: define + upgrade + whenDefined before assertions
// 13. ::part(): expose internals for external styling where appropriate
// 14. Lazy-load: guard with customElements.get() before define()
// 15. Error states: reflect error/loading/success as attributes/state

A production custom element follows a predictable API contract: documented attributes, CSS custom properties, events, slots, and proper lifecycle management.

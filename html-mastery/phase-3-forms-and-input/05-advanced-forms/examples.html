Examples 3.5 — Advanced Forms

BASIC (1–13)

1. output element

<input type="range" id="slider" min="0" max="100" value="50"
       oninput="result.value = slider.value">
<output id="result" for="slider">50</output>

output displays the result of a calculation. The for attribute links it to input elements.

2. progress element

<progress id="file-progress" value="32" max="100">32%</progress>

progress shows task completion. Without value it shows an indeterminate state. Fallback text inside for old browsers.

3. Indeterminate progress bar

<progress id="loading"></progress>
<!-- No value attribute = indeterminate (animated) -->

Omitting the value attribute creates an animated "unknown duration" progress bar.

4. meter element

<meter value="0.6" min="0" max="1">60%</meter>

meter represents a scalar measurement in a known range. Unlike progress, it has semantic coloring based on low/high/optimum.

5. meter with low, high, optimum

<meter value="0.3" min="0" max="1" low="0.25" high="0.75" optimum="0.5">
  30% capacity
</meter>

low/high partition the range into zones. optimum sets the "good" zone. Browser shows yellow or red for non-optimal values.

6. form attribute — external control

<form id="order-form" action="/order" method="post">
  <input type="text" name="item" placeholder="Item name">
</form>

<!-- Outside the form element: -->
<input type="number" name="qty" form="order-form" placeholder="Qty">
<button type="submit" form="order-form">Order</button>

The form attribute associates any control or button with a form by its id, even when placed outside the form element.

7. datalist for autocomplete

<input list="browsers" name="browser" placeholder="Choose browser">
<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
</datalist>

datalist provides autocomplete suggestions. The list attribute on input references the datalist id. Users can still type custom values.

8. datalist with labels

<input list="airports" name="departure">
<datalist id="airports">
  <option value="LHR">London Heathrow</option>
  <option value="JFK">New York JFK</option>
  <option value="CDG">Paris Charles de Gaulle</option>
</datalist>

option elements inside datalist can have both a value and a label. The label shows in the dropdown; the value is submitted.

9. novalidate attribute

<form action="/submit" novalidate>
  <input type="email" name="email" required>
  <button type="submit">Submit</button>
</form>

novalidate disables browser built-in form validation, allowing custom validation logic instead.

10. formnovalidate on submit button

<form action="/checkout">
  <input type="email" name="email" required>
  <button type="submit">Complete Purchase</button>
  <button type="submit" formnovalidate formaction="/save-draft">Save Draft</button>
</form>

formnovalidate on a specific button skips validation only for that button's submission.

11. formaction — button-specific action

<form action="/save">
  <input type="text" name="title">
  <button type="submit">Save</button>
  <button type="submit" formaction="/publish">Publish</button>
  <button type="submit" formaction="/delete" formmethod="post">Delete</button>
</form>

formaction, formmethod, formenctype can override form attributes per button.

12. input pattern attribute

<input type="text" name="postcode" pattern="[A-Z]{1,2}[0-9]{1,2}[A-Z]? [0-9][A-Z]{2}"
       title="UK postcode (e.g., SW1A 1AA)" placeholder="SW1A 1AA">

pattern applies a regular expression constraint. title provides the validation message shown to the user.

13. minlength and maxlength

<input type="text" name="username" minlength="3" maxlength="20"
       pattern="[a-z0-9_]+" title="3–20 lowercase letters, numbers, or underscores">

minlength enforces minimum character count. maxlength limits maximum. Both trigger built-in validation messages.

INTERMEDIATE (14–26)

14. Constraint Validation API

const input = document.getElementById('email');
console.log(input.validity.valid);       // true/false
console.log(input.validity.valueMissing); // required missing
console.log(input.validity.typeMismatch); // wrong type (e.g., bad email)
console.log(input.validity.patternMismatch); // pattern failed
console.log(input.validationMessage);    // browser's error message

The validity object reports which constraint failed. Use for custom validation UI.

15. setCustomValidity

const pw = document.getElementById('password');
const confirm = document.getElementById('confirm-pw');
confirm.addEventListener('input', () => {
  if (confirm.value !== pw.value) {
    confirm.setCustomValidity('Passwords do not match');
  } else {
    confirm.setCustomValidity(''); // clear error
  }
});

setCustomValidity sets a custom error message. Pass an empty string to clear the error.

16. reportValidity()

const form = document.getElementById('my-form');
document.getElementById('check-btn').addEventListener('click', () => {
  if (!form.reportValidity()) {
    console.log('Form has errors — shown to user');
  }
});

reportValidity() triggers the browser's built-in validation UI without submitting the form.

17. checkValidity() on individual input

const emailInput = document.getElementById('email');
if (!emailInput.checkValidity()) {
  // Show custom error UI
  emailInput.classList.add('invalid');
  document.getElementById('email-error').textContent = emailInput.validationMessage;
}

checkValidity() returns false (and fires an invalid event) if the input fails validation, without showing browser UI.

18. Custom validation on submit

form.addEventListener('submit', e => {
  e.preventDefault();
  let valid = true;
  form.querySelectorAll('[required]').forEach(field => {
    if (!field.value.trim()) {
      field.classList.add('error');
      valid = false;
    } else {
      field.classList.remove('error');
    }
  });
  if (valid) form.submit();
});

Custom submit handler: prevent default, validate each field manually, submit if all pass.

19. FormData API

form.addEventListener('submit', async e => {
  e.preventDefault();
  const data = new FormData(form);
  // Access individual values:
  console.log(data.get('email'));
  // Append extra fields:
  data.append('timestamp', Date.now());
  // Send with fetch:
  await fetch('/api/submit', { method: 'POST', body: data });
});

FormData automatically collects all named form controls. Works with fetch for AJAX form submission.

20. FormData iteration

const data = new FormData(form);
// Iterate all entries:
for (const [name, value] of data.entries()) {
  console.log(`${name}: ${value}`);
}
// Convert to plain object:
const obj = Object.fromEntries(data.entries());

FormData is iterable. Use entries() to loop, or Object.fromEntries() to convert to a plain object.

21. File upload with FormData

<input id="file-input" type="file" name="avatar" accept="image/*">

<script>
document.getElementById('upload-btn').addEventListener('click', async () => {
  const file = document.getElementById('file-input').files[0];
  const data = new FormData();
  data.append('avatar', file);
  data.append('userId', '42');
  await fetch('/upload', { method: 'POST', body: data });
});
</script>

FormData handles file uploads natively. Don't set Content-Type header — fetch will set it with the correct boundary.

22. Encoding JSON data as form field

const form = document.getElementById('my-form');
const metadata = { version: 2, tags: ['html', 'forms'] };
const hiddenInput = document.createElement('input');
hiddenInput.type = 'hidden';
hiddenInput.name = 'metadata';
hiddenInput.value = JSON.stringify(metadata);
form.appendChild(hiddenInput);

Store structured data in hidden inputs by JSON-stringifying. Parse on the server.

23. input event vs change event

// input fires on every keystroke / drag:
slider.addEventListener('input', () => output.value = slider.value);

// change fires when value is committed (blur, Enter):
textInput.addEventListener('change', () => console.log('Committed:', textInput.value));

// For real-time validation use input; for final validation use change or blur

input fires continuously; change fires once when the user finishes. Use the right one for your UX.

24. autocomplete attribute

<input type="text" name="name" autocomplete="name">
<input type="email" name="email" autocomplete="email">
<input type="tel" name="phone" autocomplete="tel">

<input type="text" name="cc-number" autocomplete="cc-number">

autocomplete tokens tell browsers what type of data to autofill. Use WHATWG tokens for best browser support.

25. inputmode for mobile keyboard

<input type="text" inputmode="numeric" pattern="[0-9]*" placeholder="PIN">

26. enterkeyhint for mobile

<input type="search" enterkeyhint="search" placeholder="Search...">
<input type="text" enterkeyhint="next" placeholder="First name">

enterkeyhint customizes the mobile keyboard's Enter key label. Values: enter, done, go, next, previous, search, send.

37. Readonly vs disabled

<!-- readonly: submitted with form, user can still focus/select -->
<input type="text" name="order-id" value="ORD-12345" readonly>

<!-- disabled: NOT submitted, visually greyed out -->
<input type="text" name="promo" value="SUMMER20" disabled>

<!-- To submit disabled value: use a hidden field -->
<input type="hidden" name="promo" value="SUMMER20">

readonly prevents editing but submits the value. disabled prevents editing AND submission.

ADVANCED (39–50)

39. Form-associated custom elements

class StarRating extends HTMLElement {
  static formAssociated = true;
  #internals;
  constructor() {
    super();
    this.#internals = this.attachInternals();
  }
  set value(v) { this.#internals.setFormValue(v); }
  get value() { return this.#internals.form?.elements[this.name]?.value; }
  get validity() { return this.#internals.validity; }
}
customElements.define('star-rating', StarRating);

Form-associated custom elements participate in form submission and validation via ElementInternals.

40. Validation with ElementInternals

class MyInput extends HTMLElement {
  static formAssociated = true;
  #internals = this.attachInternals();
  validate() {
    if (!this.value) {
      this.#internals.setValidity(
        { valueMissing: true },
        'This field is required',
        this.shadowRoot.querySelector('input')
      );
    } else {
      this.#internals.setValidity({});
    }
  }
}

setValidity() on ElementInternals integrates custom elements into the browser's constraint validation system.

41. Debounced validation

let debounceTimer;
emailInput.addEventListener('input', () => {
  clearTimeout(debounceTimer);
  debounceTimer = setTimeout(() => {
    fetch('/api/check-email?email=' + encodeURIComponent(emailInput.value))
      .then(r => r.json())
      .then(data => {
        emailInput.setCustomValidity(data.taken ? 'Email already registered' : '');
        emailInput.reportValidity();
      });
  }, 500);
});

Debounce async validation to avoid excessive API calls while the user is typing.

43. Progress with fetch streaming

const response = await fetch('/large-file.bin');

44. Constraint Validation API — all states

const v = input.validity;
// v.valueMissing   — required but empty
// v.typeMismatch   — wrong type (e.g., email)
// v.patternMismatch — pattern regex failed
// v.tooShort       — shorter than minlength
// v.tooLong        — longer than maxlength
// v.rangeUnderflow — less than min
// v.rangeOverflow  — greater than max
// v.stepMismatch   — not a multiple of step
// v.badInput       — browser can't convert value to type
// v.customError    — setCustomValidity was called
// v.valid          — all constraints passed

Every validity flag and what triggers it. Build precise custom UIs targeting specific failures.

45. Live form validation pattern

function validateField(field) {
  const errEl = document.getElementById(field.id + '-error');
  if (!errEl) return;
  if (field.validity.valid) {
    field.removeAttribute('aria-invalid');
    errEl.textContent = '';
    errEl.hidden = true;
  } else {
    field.setAttribute('aria-invalid', 'true');
    errEl.textContent = field.validationMessage;
    errEl.hidden = false;
  }
}
form.querySelectorAll('input, select, textarea').forEach(f => {
  f.addEventListener('blur', () => validateField(f));
  f.addEventListener('input', () => { if (f.dataset.touched) validateField(f); });
  f.addEventListener('blur', () => { f.dataset.touched = 'true'; }, { once: true });
});

Validate on blur (first interaction), then live as user types. Set aria-invalid for screen readers.

46. form.elements collection

const form = document.getElementById('survey');

const age = form.elements['age'].value;
// Access by index:
const first = form.elements[0];
// Iterate all controls:
Array.from(form.elements).forEach(el => {

form.elements is an HTMLFormControlsCollection of all controls in document order. Access by name or index.

47. requestSubmit() vs submit()

// submit() — bypasses validation and submit event listeners:
form.submit();

// requestSubmit() — triggers validation and fires submit event:
form.requestSubmit();

// requestSubmit with a specific button (to get its formaction, name/value):
form.requestSubmit(document.getElementById('publish-btn'));

Always use requestSubmit() for programmatic submission — it respects novalidate and fires the submit event.

50. Reactive form state with Proxy

const state = new Proxy({ name: '', email: '' }, {
  set(obj, prop, value) {
    obj[prop] = value;
    // Sync to DOM:
    const input = document.querySelector(`[name="${prop}"]`);
    if (input) input.value = value;
    // Update output/meter/progress:
    updatePreview(obj);
    return true;
  }
});
// Bind inputs to state:
form.querySelectorAll('input').forEach(input => {
  input.addEventListener('input', () => state[input.name] = input.value);
});

Proxy-based two-way binding: DOM changes update state; state changes update DOM. Simulates React/Vue reactivity in vanilla JS.

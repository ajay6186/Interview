Landmark Roles — 50 Examples

1. The 8 ARIA landmark roles

banner       — page header (one per page)
navigation   — set of navigation links
main         — primary content (one per page)
complementary— supporting content (aside)
contentinfo  — page footer (one per page)
search       — search functionality
form         — a significant form (needs label)
region       — named section of a page

2. Native elements with implicit landmark roles

<header>  → banner (when top-level, not inside article/section)
<nav>     → navigation
<main>    → main
<aside>   → complementary
<footer>  → contentinfo (when top-level)
<form>    → form (when it has an accessible name)

3. banner — page header

<header role="banner">
  <a href="/"><img src="logo.svg" alt="Company Logo"></a>
  <nav aria-label="Primary">...</nav>
</header>

4. navigation — labelled nav

<nav role="navigation" aria-label="Main">
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

5. main — one per page

<main id="main-content" role="main">
  <h1>Page Heading</h1>
  <!-- primary content here -->
</main>

6. complementary — sidebar

<aside role="complementary" aria-label="Related articles">
  <h2>Related Articles</h2>
  <ul>...</ul>
</aside>

7. contentinfo — page footer

<footer role="contentinfo">
  <p>© 2026 Company Inc.</p>
  <nav aria-label="Footer">...</nav>
</footer>

8. search landmark

<form role="search" aria-label="Site search">
  <input type="search" name="q" aria-label="Search query">
  <button type="submit">Search</button>
</form>

9. form landmark (requires accessible name)

<form role="form" aria-label="Newsletter signup">
  <label for="email">Email</label>
  <input id="email" type="email">
  <button>Subscribe</button>
</form>

10. region landmark (requires accessible name)

<section role="region" aria-label="Recent activity">
  <h2>Recent Activity</h2>
  <ul>...</ul>
</section>
A <section> only becomes a landmark when it has an accessible name via aria-label or aria-labelledby.

11. Labelling landmarks with aria-labelledby

<nav aria-labelledby="nav-heading">
  <h2 id="nav-heading">Categories</h2>
  <ul>...</ul>
</nav>

12. Multiple nav elements — must be distinguished

<nav aria-label="Primary navigation">...</nav>
<nav aria-label="Breadcrumb">...</nav>
<nav aria-label="Footer navigation">...</nav>

13. Multiple aside elements

<aside aria-label="Author bio">...</aside>
<aside aria-label="Advertisement">...</aside>

14. header inside article — not a banner

<article>
  <header> <!-- NOT a banner landmark -->
    <h2>Article Title</h2>
    <p>By Author</p>
  </header>
  <p>Content...</p>
</article>

15. footer inside article — not contentinfo

<article>
  <p>Content...</p>
  <footer> <!-- NOT a contentinfo landmark -->
    <p>Published: 2026-01-01</p>
  </footer>
</article>

16. Full page landmark structure

<body>
  <header role="banner">...</header>
  <nav role="navigation" aria-label="Main">...</nav>
  <main role="main">
    <h1>...</h1>
    ...
  </main>
  <aside role="complementary">...</aside>
  <footer role="contentinfo">...</footer>
</body>

17. Nested navigation inside main

<main>
  <nav aria-label="Table of contents">
    <a href="#section1">Section 1</a>
    <a href="#section2">Section 2</a>
  </nav>
  <article>...</article>
</main>

18. Breadcrumb navigation

<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li aria-current="page">Widget</li>
  </ol>
</nav>

19. Pagination navigation

<nav aria-label="Pagination">
  <a href="?page=1" aria-label="Previous page">←</a>
  <a href="?page=2" aria-current="page">2</a>
  <a href="?page=3" aria-label="Next page">→</a>
</nav>

20. Skip link targets landmarks

<a href="#main-content">Skip to main content</a>
<main id="main-content" tabindex="-1">...</main>

21. aria-current for current page in nav

<nav aria-label="Main">
  <a href="/" aria-current="page">Home</a>
  <a href="/about">About</a>
</nav>

22. Site map landmark structure (two-column layout)

<header>...logo, primary nav...</header>
<div class="layout">
  <main>...primary content...</main>
  <aside>...secondary sidebar...</aside>
</div>
<footer>...copyright, secondary nav...</footer>

23. Don't use role="main" twice

<!-- WRONG -->
<main>...</main>
<div role="main">...</div> <!-- two main landmarks confuse users -->

<!-- RIGHT — only one main per page -->
<main>...</main>

24. Document has all required landmarks

// Minimum recommended set:
// ✓ banner (header)
// ✓ main
// ✓ contentinfo (footer)
// Optional but useful:
// - navigation
// - search (if site has search)

25. All content inside a landmark

<!-- All visible content should be inside some landmark -->
<body>
  <!-- no orphaned text here -->
  <header>...</header>
  <main>...all content inside...</main>
  <footer>...</footer>
</body>

26. SPA — update landmarks on route change

// After navigation, main content updates
// Screen readers can re-read from main landmark
router.on('navigate', () => {
  document.getElementById('main-content').focus();
});

27. Landmark roles in SVG context

<svg role="img" aria-label="Sales chart for Q1 2026">
  <title>Sales chart for Q1 2026</title>
  ...
</svg>

28. Don't add redundant role to native element

<!-- Redundant — nav already has navigation role -->
<nav role="navigation">...</nav>

<!-- But it doesn't hurt anything and aids cross-browser compat -->

29. header/footer in sectioning elements lose landmark role

<section>
  <header> <!-- generic, no landmark role -->
    <h2>Section heading</h2>
  </header>
</section>

30. article — not a landmark, but semantically useful

<article>
  <h2>Blog Post Title</h2>
  <p>Self-contained content...</p>
</article>
article is not an ARIA landmark, but it is announced by screen readers as a region.

31. Visible heading links landmarks to page structure

<main>
  <h1>Page Title</h1> <!-- heading connects to main landmark -->
  ...
</main>

32. Aside — advertising disclaimer

<aside aria-label="Sponsored content">
  <p>Advertisement</p>
  <!-- ad content -->
</aside>

33. Complex app — multiple main-like regions

<!-- Use role="region" with labels instead of multiple mains -->
<main>
  <section aria-label="Inbox">...</section>
  <section aria-label="Message detail">...</section>
</main>

34. Toolbar — not a landmark but a composite widget

<div role="toolbar" aria-label="Text formatting">
  <button>Bold</button>
  <button>Italic</button>
</div>

35. Application role — only for complex widget areas

<!-- Rarely needed; disables most screen reader shortcuts -->
<div role="application" aria-label="Spreadsheet editor">
  ...
</div>
Only use role="application" when you fully implement all keyboard interaction yourself.

39. Headings within landmarks

<!-- Each major landmark should have a heading -->
<nav aria-label="Products">
  <h2>Products</h2> <!-- or use aria-labelledby -->
  ...
</nav>

40. Landmark order matters

<!-- Logical reading order -->
<header>...</header>
<nav>...</nav>     <!-- before main, so it appears in landmark list first -->
<main>...</main>
<footer>...</footer>

41. Sticky navigation — keep it accessible

<nav id="sticky-nav" aria-label="Main" style="position:sticky;top:0">
  ...
</nav>
<!-- Sticky nav shouldn't cover focused elements; use scroll-padding-top -->

42. scroll-padding-top for sticky nav

html {
  scroll-padding-top: 60px; /* height of sticky nav */
}

43. Mobile navigation — aria-expanded pattern

<button aria-expanded="false" aria-controls="mobile-nav">Menu</button>
<nav id="mobile-nav" aria-label="Main" hidden>
  ...
</nav>

44. Side navigation drawer

<nav aria-label="Side navigation"
     aria-hidden="false"
     id="sidenav">
  ...
</nav>

<!-- When closed: aria-hidden="true" and move focus back to toggle -->

45. Don't landmark every div

<!-- TOO MANY LANDMARKS — overwhelming for screen reader users -->
<section aria-label="Intro paragraph"><p>...</p></section>
<section aria-label="Second paragraph"><p>...</p></section>

<!-- Better: one meaningful section -->
<section aria-label="Introduction">
  <p>...</p>
  <p>...</p>
</section>

46. Hidden landmark when sidebar is closed

// When sidebar is hidden, remove it from landmark list
sidebar.setAttribute('aria-hidden', 'true');
// When sidebar opens
sidebar.removeAttribute('aria-hidden');

47. Map of ARIA roles to HTML elements

banner        → <header> (top-level)
navigation    → <nav>
main          → <main>
complementary → <aside>
contentinfo   → <footer> (top-level)
search        → no native element
form          → <form> (if named)
region        → <section> (if named)

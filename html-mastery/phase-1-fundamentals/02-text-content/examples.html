Examples 1.2 — Text Content

BASIC (1–13)

1. All six heading levels

Heading 1 — Page Title

Heading 2 — Section

Heading 3 — Subsection

Heading 4

Heading 5
Heading 6

h1 is largest and most important. Never skip levels (e.g., h1 → h3).

2. Paragraph element

This is a paragraph. Paragraphs are block-level elements separated by whitespace above and below.

A second paragraph. The browser adds margin between them automatically.

<p> is the most common block element. Never use <br> to fake paragraph spacing.

3. strong — important text

Please read the terms and conditions before proceeding.

<strong> conveys importance or urgency. Screen readers may add emphasis. Renders bold by default.

4. em — stressed emphasis

I really mean it — this is important.

She said she did finish the homework.

<em> indicates vocal stress. Renders italic by default. Contrast with <i> which is just typographic italic.

5. mark — highlighted text

Your search for "semantic HTML" returned 12 results.

<mark> represents text highlighted for reference, like search results. Yellow background by default.

6. del and ins — edits

The meeting is on Tuesday Wednesday.

Price: $99 $79

<del> = removed content (strikethrough). <ins> = inserted content (underline). Both accept datetime attributes.

7. abbr — abbreviations

HTML was created by Tim Berners-Lee.

CSS controls the visual presentation.

<abbr title="..."> provides the full expansion. Hover over it to see the tooltip.

8. code — inline code

Use Array.prototype.map() to transform array elements.

The color property accepts hex, rgb, or named values.

<code> is for inline code fragments. Use <pre><code> for blocks. Renders in monospace.

9. pre — preformatted text

const add = (a, b) => a + b;
console.log(add(2, 3)); // 5

<pre> preserves whitespace and line breaks. Ideal for code blocks. Renders in monospace.

10. kbd — keyboard input

Press Ctrl + C to copy selected text.

Save your file with Cmd + S on Mac.

<kbd> represents keyboard input. Useful in documentation. Renders in monospace.

11. blockquote

The Web does not just connect machines, it connects people.

<blockquote> is for extended quotations. The cite attribute (URL) credits the source. Add <cite> for the author name.

12. br — line break

123 Main Street
Suite 4B
Springfield, IL 62701

<br> inserts a line break within flow content. Use sparingly — only where the break is semantically meaningful (addresses, poetry).

13. hr — thematic break

End of Chapter 1.

Chapter 2 begins here.

<hr> is a thematic break between sections, not just a decorative line. Renders as a horizontal rule.

INTERMEDIATE (14–26)

14. samp — sample output

After running the script, you should see: Build succeeded in 2.4s

<samp> represents sample output from a program. Often used with <kbd> in terminal documentation.

15. var — variable name

Set x = 5 and y = 10, then compute x + y.

<var> marks a variable in a mathematical expression or programming context. Renders italic by default.

16. sub and sup — subscript and superscript

Water is H2O.

Einstein's famous equation: E = mc2

The 1st of January is a holiday.

<sub> and <sup> are for mathematical formulas, footnote references, and chemical formulas.

17. small — fine print

Buy now for $9.99/month. Billed annually. Cancel anytime.

<small> represents side comments, fine print, or copyright notices. Renders at a smaller font size.

18. b vs strong — typographic bold vs important

Product name: Ultra Widget Pro

Warning: This product contains nuts.

<b> is purely typographic (draw attention without importance). <strong> conveys semantic importance. Both render bold.

19. i vs em — typographic italic vs emphasis

The ship was named HMS Victory.

You must read the safety instructions.

<i> is for technical terms, foreign words, ship names, thoughts. <em> is for spoken emphasis. Both render italic.

20. blockquote with cite element

<cite> marks the title of a work or author name. The <figure>/<figcaption> pattern is the recommended way to attribute quotes.

21. del with datetime attribute

The deadline was January 15th February 1st.

The datetime attribute on <del> and <ins> records when the change was made in ISO 8601 format.

22. dfn — defining instance

The HTML (HyperText Markup Language) is the standard language for web pages.

<dfn> marks the defining instance of a term. Often paired with an <abbr> inside it.

23. q — inline quotation

Tim Berners-Lee said The Web is for everyone.

<q> is for short inline quotations. Browsers add quotation marks automatically. Use <blockquote> for longer quotes.

24. u — underline for non-emphasis

This text has a spell-checking annotation or Chinese proper noun marker.

<u> is for annotations that don't imply emphasis — like spelling errors or proper nouns in certain languages. Not for hyperlinks.

25. data element with machine-readable value

The product costs $49.99.

Temperature: 37.2°C

<data> links human-readable content with a machine-readable value attribute. For time values, prefer <time>.

26. wbr — word break opportunity

This_is_a_very_long_identifier_that_might_need_a_break_point_added.

<wbr> suggests a possible line-break location for the browser. Useful for long compound words or URLs.

NESTED (27–38)

27. pre + code block

function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

<pre><code> is the correct pattern for multi-line code blocks. Note escaped < as &lt;.

28. Nested emphasis

This is very important information that you should not ignore.

You can nest <em> inside <strong> (or vice versa) for combined emphasis.

29. Blockquote with nested structure

Good code is its own best documentation.

As you're about to add a comment, ask yourself:

How can I improve the code so that this comment isn't needed?

— Steve McConnell, Code Complete

Blockquotes can contain paragraphs, nested quotes, and a footer for attribution.

33. Address element

Written by Jane Doe.
Visit us at:
123 Main Street
Springfield, IL 62701

<address> provides contact information for its nearest <article> or <body> ancestor. Not for generic postal addresses.

34. time element with datetime

Published on March 24, 2026.

The event starts at 9 AM on April 1st.

Duration: 2 hours and 30 minutes.

<time> provides machine-readable datetime in ISO 8601 format while showing human-friendly text.

35. Mixing del, ins, and mark

The project deadline is Q1 2026 Q2 2026. The most critical milestone is the design review.

Combine editing markup (<del>/<ins>) with <mark> highlights in document revision workflows.

36. Poetry with br

HTML is semantic,
And so should you.

Use <br> inside <p> for verse/poetry where line breaks are meaningful. Alternatively use <br> inside a dedicated poem container.

37. Figure with code caption

<figure> can wrap code listings, giving them a <figcaption> for cross-reference.

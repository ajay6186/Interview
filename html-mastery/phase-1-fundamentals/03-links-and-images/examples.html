Examples 1.3 — Links & Images

BASIC (1–13)

1. Basic hyperlink

Visit Example.com
<a href="https://www.example.com">Visit Example.com</a>

The href attribute specifies the URL destination. Absolute URLs include the protocol.

2. Relative link

About Page
<a href="about.html">About Page</a>
<a href="../index.html">Home (one level up)</a>
<a href="/contact">Contact (from root)</a>

Relative URLs are resolved relative to the current page URL. Prefer them for internal links.

3. Link opening in new tab

Open in new tab
<a href="https://www.example.com"
   target="_blank"
   rel="noopener noreferrer">Open in new tab</a>

Always add rel="noopener noreferrer" with target="_blank" to prevent tabnapping attacks.

4. Anchor link (in-page navigation)

Jump to section
<a href="#section-id">Jump to section</a>
...
<section id="section-id">...</section>

Fragment identifiers (#id) scroll the page to the matching element.

5. Email link

hello@example.com
<a href="mailto:hello@example.com">hello@example.com</a>
<a href="mailto:support@example.com?subject=Help&body=I%20need%20help">Email Support</a>

mailto: links open the user's default email client. Use URL encoding for subject/body.

6. Phone link

+1 (555) 123-4567
<a href="tel:+15551234567">+1 (555) 123-4567</a>

tel: links allow mobile users to tap to call. Use E.164 format (+countrycode...).

7. Download link

Download Report (PDF)
<a href="report.pdf" download>Download Report (PDF)</a>
<a href="data.csv" download="export-2026.csv">Download CSV</a>

The download attribute triggers a download instead of navigation. Optionally specify a filename.

8. Basic image

<img src="photo.jpg" alt="A grey placeholder image" width="150" height="100">

Always include alt, width, and height. Width/height prevent layout shift (CLS).

9. Decorative image with empty alt

<img src="decorative-border.png" alt="">

Empty alt="" tells screen readers to skip the image entirely — correct for purely decorative images.

10. Lazy-loaded image

<img src="large-photo.jpg"
     alt="Mountain landscape at sunset"
     width="800" height="600"
     loading="lazy">

loading="lazy" defers loading until the image is near the viewport. Do NOT lazy-load above-the-fold (LCP) images.

11. Image with explicit dimensions for CLS

<img src="hero.jpg"
     alt="Team members at the office"
     width="1200"
     height="630">

Setting width and height lets the browser reserve space before the image loads, preventing cumulative layout shift.

12. Image as a link

<a href="https://www.example.com">
  <img src="logo.png" alt="Acme Co — go to homepage" width="120" height="40">
</a>

When an image is the sole content of a link, the alt text must describe the link's destination.

13. Link with title attribute

Glossary
<a href="glossary.html" title="Opens the full glossary of terms">Glossary</a>

The title attribute shows a tooltip on hover. It is NOT a substitute for clear link text.

INTERMEDIATE (14–26)

14. Skip navigation link

<a href="#main-content" class="skip-link">Skip to main content</a>
<nav>...</nav>
<main id="main-content">...</main>

<style>
  .skip-link { position: absolute; top: -40px; }
  .skip-link:focus { top: 0; }
</style>

A skip link lets keyboard users bypass repetitive navigation. Essential for accessibility (WCAG 2.4.1).

15. Link with aria-label for icon buttons

<a href="/notifications" aria-label="View 3 notifications">
  <svg aria-hidden="true">...</svg>
</a>

When a link contains only an icon, add aria-label to describe the destination for screen reader users.

16. Image crossorigin for canvas use

<img src="https://cdn.example.com/photo.jpg"
     alt="Cross-origin image"
     crossorigin="anonymous">

crossorigin="anonymous" is required when drawing cross-origin images to a canvas, otherwise the canvas is tainted and data can't be read.

17. Responsive image with max-width

<img src="photo.jpg" alt="Landscape" style="max-width: 100%; height: auto;">

The CSS pattern max-width: 100%; height: auto; is the classic way to make images scale within their container.

18. Image with decoding="async"

<img src="large-image.jpg"
     alt="Large illustration"
     decoding="async"
     loading="lazy">

decoding="async" allows the browser to decode the image off the main thread, reducing jank.

19. Figure with figcaption

<figure>
  <img src="chart.png" alt="Chart showing quarterly revenue" width="800" height="400">
  <figcaption>Figure 1: Quarterly revenue for 2026.</figcaption>
</figure>

<figure>/<figcaption> groups an image with its caption. The caption supplements (not duplicates) the alt text.

20. Link with rel="nofollow"

<a href="https://sponsor.example.com" rel="nofollow sponsored">Our Sponsor</a>
<a href="https://user-submitted.example.com" rel="nofollow ugc">User Link</a>

rel="nofollow" tells search engines not to pass PageRank. "sponsored" and "ugc" (user-generated content) are specialized variants.

21. Link to specific page with hreflang

<a href="/fr/accueil" hreflang="fr">Version française</a>
<a href="/de/startseite" hreflang="de">Deutsche Version</a>

hreflang on <a> indicates the language of the linked resource. Used for language switchers.

22. Link with referrerpolicy

<a href="https://example.com"
   referrerpolicy="no-referrer"
   target="_blank"
   rel="noopener">Private Link</a>

referrerpolicy controls how much referrer info is sent. "no-referrer" sends nothing; useful for privacy-sensitive links.

23. Image with fetchpriority

<!-- Hero/LCP image — load it fast -->
<img src="hero.jpg" alt="Hero banner" fetchpriority="high" width="1200" height="630">

<!-- Below-fold image — deprioritize -->
<img src="thumbnail.jpg" alt="Article thumbnail" fetchpriority="low" loading="lazy">

fetchpriority hints allow you to boost or reduce image load priority relative to the browser's default heuristics.

24. Anchor with ping attribute

<a href="https://example.com" ping="https://analytics.mysite.com/track">
  Tracked Link
</a>

ping sends a POST request to the listed URLs when the link is followed. Used for analytics. Some browsers block it.

25. SMS link

<a href="sms:+15551234567">Send SMS</a>
<a href="sms:+15551234567?body=Hello%20there">Pre-filled SMS</a>

sms: links open the SMS app on mobile devices. Useful for contact pages and support flows.

26. image loading="eager" for LCP

<!-- Above-the-fold hero image — do NOT lazy load -->
<img src="hero.webp"
     alt="Product hero shot"
     width="1200" height="600"
     loading="eager"
     fetchpriority="high">

loading="eager" is the default. Be explicit for hero images to counter any framework that mass-applies lazy loading.

NESTED (27–38)

27. Navigation with logo and links

Products About Contact

Logo linked to home page. The alt text of the logo image describes the link destination.

28. Card with image and link

Article Title

Short description of the article content.

Card pattern: the thumbnail image link has empty alt (the heading link already names the destination).

29. Image gallery grid

Gallery grid: each image wrapped in a link. Each has a descriptive alt text.

30. Breadcrumb navigation with links

  1. Home
  2. /
  3. Products
  4. /
  5. Widget Pro

Breadcrumb navigation pattern with aria-label and aria-current="page" on the last item.

31. Link with external icon

Example.com ↗ (opens in new tab)

Visually indicate external links with an icon, and add screen-reader-only text to announce the new-tab behavior.

32. Image map (usemap)

<img src="world-map.png" alt="World map — click a continent" usemap="#worldmap" width="600" height="300">
<map name="worldmap">
  <area shape="rect" coords="100,50,200,150" href="/europe" alt="Europe">
  <area shape="circle" coords="350,150,60" href="/asia" alt="Asia">
  <area shape="poly" coords="80,200,120,180,160,220" href="/africa" alt="Africa">
</map>

Image maps allow clickable regions on an image. Consider SVG for better accessibility and responsiveness.

33. Progressive image loading (blur-up)

<img
  src="placeholder-tiny.jpg"
  data-src="full-size.jpg"
  alt="Scenic landscape"
  class="lazy"
  width="1200"
  height="800">

<script>
  const observer = new IntersectionObserver(entries => {
    entries.forEach(e => {
      if (e.isIntersecting) {
        e.target.src = e.target.dataset.src;
        observer.unobserve(e.target);
      }
    });
  });
  document.querySelectorAll("img.lazy").forEach(img => observer.observe(img));
</script>

Custom lazy loading with IntersectionObserver for browsers that don't support loading="lazy" or for custom effects.

34. Download link with Content-Disposition fallback info

<!-- download attribute works for same-origin or data URLs -->
<a href="/generate-report?format=csv" download="report-2026-q1.csv">
  Download Q1 2026 Report (CSV)
</a>

<!-- Cross-origin: rely on Content-Disposition header instead -->
<a href="https://cdn.example.com/data.csv">
  Download Data (CSV)
</a>

The download attribute only works for same-origin URLs; cross-origin requires a Content-Disposition: attachment response header.

35. Link list as navigation menu

  • Home
  • About
  • Services
  • Blog
  • Contact

Navigation menus should use <nav> + <ul><li><a> for semantic structure.

36. Inline SVG as link content

<a href="/" aria-label="Home">
  <svg width="24" height="24" viewBox="0 0 24 24" aria-hidden="true">
    <path d="M10 20v-6h4v6h5v-8h3L12 3 2 12h3v8z"/>
  </svg>
</a>

SVG icons in links: aria-hidden on the SVG (decorative), aria-label on the link to name the destination.

37. Image with object-fit CSS

object-fit: cover fills the container while maintaining aspect ratio and cropping edges — ideal for thumbnails.

38. Data URI image (inline)

<!-- Tiny inline PNG (base64 encoded) -->
<img
  src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mNkYPhfDwAChwGA60e6kgAAAABJRU5ErkJggg=="
  alt="1x1 pixel tracker"
  width="1" height="1">

Data URI images are embedded inline as base64. Useful for very small images to avoid HTTP requests. Large images should remain external files.

ADVANCED (39–50)

39. Anchor targeting iframe

<a href="page.html" target="preview-frame">Preview</a>
<iframe name="preview-frame" src="about:blank" title="Preview pane"></iframe>

The target attribute can name an iframe. Clicking the link loads the URL into that iframe.

40. Pre-connected link for faster image loading

<head>
  <link rel="preconnect" href="https://images.cdn.example.com">
</head>
<body>
  <img src="https://images.cdn.example.com/photo.jpg" alt="CDN hosted photo">
</body>

preconnect establishes the TCP/TLS connection to the CDN early, reducing image load latency.

41. Link with data-* attribute for JS hooks

<a href="/product/42"
   data-product-id="42"
   data-track="product-click"
   class="product-link">
  Buy Now
</a>

<script>
  document.querySelectorAll(".product-link").forEach(link => {
    link.addEventListener("click", e => {
      analytics.track(link.dataset.track, { id: link.dataset.productId });
    });
  });
</script>

data-* attributes store metadata for JavaScript without polluting the DOM with non-standard attributes.

42. Responsive art direction with picture

<picture>
  <source media="(min-width: 1024px)" srcset="hero-wide.webp" type="image/webp">
  <source media="(min-width: 768px)" srcset="hero-medium.webp" type="image/webp">
  <source srcset="hero-narrow.webp" type="image/webp">
  <img src="hero-narrow.jpg" alt="Product hero image" width="800" height="400">
</picture>

The <picture> element enables art direction: different images for different breakpoints plus format fallbacks.

43. Anchor preventing default with JavaScript

<a href="/cart" id="add-to-cart">Add to Cart</a>

<script>
  document.getElementById("add-to-cart").addEventListener("click", e => {
    e.preventDefault(); // Stop navigation
    addItemToCart();     // Custom logic
  });
</script>

Use e.preventDefault() on anchor click handlers to intercept navigation. Keep a real href for non-JS fallback and right-click "open in new tab".

44. Aspect ratio preservation with CSS

<style>
  .img-16-9 {
    aspect-ratio: 16 / 9;
    width: 100%;
    object-fit: cover;
  }
</style>
<img class="img-16-9" src="banner.jpg" alt="Event banner">

CSS aspect-ratio ensures images maintain their ratio while scaling responsively — no need for padding-top hacks.

45. Image with error fallback via onerror

<img
  src="user-uploaded.jpg"
  alt="User profile photo"
  onerror="this.src='default-avatar.png'; this.onerror=null;"
  width="64" height="64">

onerror provides a graceful fallback if an image fails to load. Set onerror=null to prevent infinite loops.

46. Link prefetch for next page

<!-- In <head> -->
<link rel="prefetch" href="/checkout">

<!-- In body -->
<a href="/checkout">Proceed to Checkout</a>

Prefetching the next likely page during idle time makes navigation feel instant. Use for predictable user flows.

47. Open Graph image meta tag

<meta property="og:image" content="https://example.com/og-image.jpg">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:image:alt" content="Screenshot of the dashboard">

OG images control the preview thumbnail on social media. Recommended size: 1200×630px. Always include og:image:alt.

48. Accessible icon link with tooltip

<style>
  .icon-btn { position: relative; }
  .icon-btn::after { content: attr(aria-label); position: absolute; bottom: 100%; left: 50%;
    transform: translateX(-50%); background: #333; color: #fff; padding: 2px 6px;
    border-radius: 3px; font-size: 12px; white-space: nowrap; opacity: 0; }
  .icon-btn:hover::after, .icon-btn:focus::after { opacity: 1; }
</style>
<a href="/settings" class="icon-btn" aria-label="Settings">⚙️</a>

CSS tooltip from aria-label: accessible, no JS needed, works on both hover and focus.

49. Image srcset with pixel density descriptors

<img
  src="logo.png"
  srcset="logo.png 1x, logo@2x.png 2x, logo@3x.png 3x"
  alt="Acme Co logo"
  width="120" height="40">

Pixel density descriptors (1x, 2x, 3x) let high-DPI (Retina) displays use sharper images without affecting layout.

50. Complete responsive image with all best practices

<figure>
  <picture>
    <source
      media="(min-width: 1024px)"
      srcset="hero-1200.webp 1200w, hero-2400.webp 2400w"
      sizes="100vw"
      type="image/webp">
    <source
      media="(min-width: 600px)"
      srcset="hero-800.webp 800w, hero-1600.webp 1600w"
      sizes="100vw"
      type="image/webp">
    <img
      src="hero-400.jpg"
      srcset="hero-400.jpg 400w, hero-800.jpg 800w"
      sizes="100vw"
      alt="Team celebrating product launch in the office"
      width="1200" height="630"
      loading="eager"
      fetchpriority="high"
      decoding="async">
  </picture>
  <figcaption>The Acme team celebrating the v3.0 launch.</figcaption>
</figure>

The gold-standard responsive image: picture + source (WebP + JPEG fallback) + srcset + sizes + alt + dimensions + loading + fetchpriority + figcaption.

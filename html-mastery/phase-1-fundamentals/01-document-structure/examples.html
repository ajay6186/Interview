Examples 1.1 — Document Structure

BASIC (1–13)

1. Minimal valid HTML5 document

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Page Title</title>
</head>
<body>
  <p>Hello, world!</p>
</body>
</html>

The smallest possible valid HTML5 document.

2. DOCTYPE declaration

<!DOCTYPE html>

Always the very first line. Tells the browser to use standards mode. Case-insensitive but lowercase is conventional.

3. html element with lang attribute

<html lang="en">

lang helps screen readers choose the correct language/accent and aids spell-checkers. Use BCP-47 codes: "en", "fr", "de", "zh-CN", etc.

4. meta charset

<meta charset="UTF-8">

Declares character encoding. UTF-8 supports every Unicode character. Must appear within the first 1024 bytes of the document.

5. meta viewport for responsive design

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Without this, mobile browsers zoom out to show a desktop-width page. width=device-width sets the viewport to the device width.

6. title element

<title>My Awesome Website — Home</title>

Shows in browser tabs, bookmarks, and search results. Keep it under ~60 characters. Format: "Page Name — Site Name".

7. meta description

<meta name="description" content="Learn HTML from basics to advanced with hands-on exercises.">

Used by search engines as the snippet under the title in search results. Aim for 150–160 characters.

8. head element — what belongs inside

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title</title>
  <meta name="description" content="...">
  <link rel="stylesheet" href="style.css">
  <script src="app.js" defer></script>
</head>

The <head> holds metadata — nothing rendered directly. CSS and deferred JS go here too.

9. body element

<body>
  <h1>Welcome</h1>
  <p>All visible content lives inside <body>.</p>
</body>

All rendered content goes inside <body>. There must be exactly one.

10. lang attribute for a different language

<html lang="ja"> <!-- Japanese -->
<html lang="ar" dir="rtl"> <!-- Arabic, right-to-left -->
<html lang="zh-Hans"> <!-- Simplified Chinese -->

Use subtags for regional variants. Add dir="rtl" for right-to-left languages.

11. Multiple meta tags in head

<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="...">
<meta name="author" content="Jane Doe">
<meta name="keywords" content="html, tutorial, beginner">

<meta> is a void element (no closing tag). Stack as many as needed.

12. link element for favicon

<link rel="icon" type="image/png" href="/favicon.png">
<link rel="apple-touch-icon" href="/apple-touch-icon.png">

Favicons appear in browser tabs and bookmarks. SVG favicons are increasingly supported.

13. script with defer vs async

<script src="app.js" defer></script>   <!-- runs after DOM parsed -->
<script src="lib.js" async></script>   <!-- runs as soon as downloaded -->
<script src="old.js"></script>         <!-- blocks parsing (avoid) -->

Prefer defer for most scripts. Use async for independent analytics or ads scripts.

INTERMEDIATE (14–26)

14. Canonical link tag

<link rel="canonical" href="https://example.com/page/">

Tells search engines which URL is the authoritative version, preventing duplicate-content penalties.

15. Theme color meta tag

<meta name="theme-color" content="#4285f4">
<meta name="theme-color" content="#ffffff" media="(prefers-color-scheme: light)">
<meta name="theme-color" content="#000000" media="(prefers-color-scheme: dark)">

Colors the browser chrome on mobile. Can be conditional on color scheme.

16. base element

<base href="https://example.com/" target="_blank">

Sets the base URL for all relative links on the page. Only one <base> allowed per document.

17. robots meta tag

<meta name="robots" content="index, follow">       <!-- default -->
<meta name="robots" content="noindex, nofollow">    <!-- block crawlers -->
<meta name="robots" content="noindex, follow">      <!-- follow links but don't index -->

Controls search engine crawling. Use noindex on staging environments, login pages, etc.

18. Content-Security-Policy via meta

<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; img-src *; script-src 'self'">

Restricts what resources the page can load, reducing XSS risk. Prefer HTTP header CSP in production.

19. http-equiv refresh redirect

<meta http-equiv="refresh" content="5; url=https://example.com/new-page">

Redirects after N seconds. Avoid in production — use a server-side 301 redirect instead.

20. preload and prefetch link hints

<link rel="preload" href="hero.jpg" as="image">
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
<link rel="prefetch" href="/next-page.html">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

Resource hints improve performance. preload is for current page needs; prefetch for future navigation.

21. DNS prefetch

<link rel="dns-prefetch" href="//fonts.googleapis.com">
<link rel="dns-prefetch" href="//cdn.example.com">

Tells the browser to resolve DNS for external domains early, reducing connection latency.

22. X-UA-Compatible for IE (legacy)

<meta http-equiv="X-UA-Compatible" content="IE=edge">

Forces IE to use its highest standards mode. Largely irrelevant now but you may see it in older codebases.

23. Language variation with hreflang

<link rel="alternate" hreflang="en" href="https://example.com/en/">
<link rel="alternate" hreflang="es" href="https://example.com/es/">
<link rel="alternate" hreflang="x-default" href="https://example.com/">

Tells search engines about language/regional variants of the same page.

24. Inline styles in head (valid but discouraged)

<head>
  <style>
    body { margin: 0; font-family: sans-serif; }
  </style>
</head>

Inline <style> is valid inside <head>. For small critical CSS it avoids a render-blocking request.

25. noscript fallback

<noscript>
  <p>Please enable JavaScript to use this website.</p>
</noscript>

<noscript> content is shown only when JS is disabled. Use sparingly — most features should degrade gracefully.

26. manifest link for PWA

<link rel="manifest" href="/manifest.json">

Links a Web App Manifest for Progressive Web App features (install prompt, splash screen, icons).

NESTED (27–38)

27. Complete production-ready head

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Product Page — Acme Co</title>
  <meta name="description" content="Buy widgets from Acme Co. Fast shipping.">
  <link rel="canonical" href="https://acme.co/products/">
  <link rel="icon" href="/favicon.svg" type="image/svg+xml">
  <link rel="manifest" href="/manifest.json">
  <meta name="theme-color" content="#0066cc">
  <link rel="stylesheet" href="/styles/main.css">
  <script src="/scripts/app.js" defer></script>
</head>

A comprehensive, ordered head element for a real production page.

28. Document with nested lang overrides

<html lang="en">
<body>
  <p>This paragraph is in English.</p>
  <p lang="fr">Ce paragraphe est en français.</p>
  <blockquote lang="de">Das ist ein deutsches Zitat.</blockquote>
</body>
</html>

lang can be overridden on any element, not just <html>.

29. HTML document with ARIA landmark roles

<body>
  <header role="banner">...</header>
  <nav role="navigation">...</nav>
  <main role="main">...</main>
  <aside role="complementary">...</aside>
  <footer role="contentinfo">...</footer>
</body>

Semantic HTML5 elements implicitly carry ARIA roles. Explicit roles add redundancy for older assistive tech.

30. Conditional comments (IE legacy pattern)

<!--[if lt IE 9]>
  <script src="html5shiv.js"></script>
<![endif]-->

Conditional comments were IE-specific. They are ignored by all modern browsers. For reference only.

31. Template document shell with Open Graph

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Article Title — Site Name</title>
  <meta property="og:title" content="Article Title">
  <meta property="og:description" content="A short summary.">
  <meta property="og:image" content="https://example.com/og-image.jpg">
  <meta property="og:url" content="https://example.com/article/">
  <meta property="og:type" content="article">
</head>

Open Graph tags control how the page looks when shared on Facebook, LinkedIn, Slack, etc.

32. Document with inline SVG favicon

<link rel="icon" href="data:image/svg+xml,<svg xmlns='http://www.w3.org/2000/svg'
  viewBox='0 0 100 100'><text y='.9em' font-size='90'>🔥</text></svg>">

A data URI SVG favicon that renders an emoji icon. No file needed.

33. Skeleton HTML with critical CSS inlined

<head>
  <meta charset="UTF-8">
  <style>
    /* Critical above-the-fold CSS */
    body { margin: 0; font-family: system-ui; }
    .hero { background: #0066cc; color: #fff; padding: 2rem; }
  </style>
  <link rel="stylesheet" href="non-critical.css" media="print" onload="this.media='all'">
</head>

The print media trick loads CSS asynchronously, avoiding a render-blocking request.

34. Document structure for an email (HTML email skeleton)

<!DOCTYPE html>
<html lang="en" xmlns="http://www.w3.org/1999/xhtml">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Email Subject Line</title>
</head>
<body>
  <!-- Email clients strip most head content; inline all styles -->
</body>
</html>

HTML email requires extra legacy support. Most CSS must be inlined.

35. Viewport meta variations

<!-- Standard responsive -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- Prevent user zoom (accessibility concern — avoid) -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">

<!-- Minimum/maximum scale -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1, maximum-scale=5">

Never use user-scalable=no — it prevents accessibility zooming.

36. Document with JSON-LD structured data

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "My Article Title",
  "author": { "@type": "Person", "name": "Jane Doe" },
  "datePublished": "2026-01-01",
  "image": "https://example.com/photo.jpg"
}
</script>

JSON-LD in the <head> provides structured data for rich search results.

37. Documenting character entity references

<p>&amp;  → & (ampersand)</p>
<p>&lt;   → < (less-than)</p>
<p>&gt;   → > (greater-than)</p>
<p>&quot; → " (double quote)</p>
<p>&apos; → ' (apostrophe)</p>
<p>&nbsp; → non-breaking space</p>
<p>&copy; → © (copyright)</p>

Character entity references let you include special characters safely in HTML.

38. Document with a comment block

<!-- This is a single-line comment -->

<!--
  This is a multi-line comment.
  Comments are NOT sent to the browser but ARE visible in page source.
  Never put passwords or secrets in HTML comments.
-->

<!-- TODO: Replace this placeholder with real content -->

HTML comments use <!-- ... -->. They cannot be nested.

ADVANCED (39–50)

39. Content-Type enforcement via meta

<meta http-equiv="content-type" content="text/html; charset=utf-8">

The HTML5 <meta charset> shorthand replaced this verbose form. You may see it in older codebases.

40. Multiple stylesheets with media queries

<link rel="stylesheet" href="base.css">
<link rel="stylesheet" href="print.css" media="print">
<link rel="stylesheet" href="narrow.css" media="(max-width: 600px)">
<link rel="stylesheet" href="dark.css" media="(prefers-color-scheme: dark)">

Conditionally load stylesheets based on media queries. Non-matching stylesheets still download but don't block rendering.

41. Integrity and crossorigin attributes

<script
  src="https://cdn.example.com/library.js"
  integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/ux..."
  crossorigin="anonymous">
</script>

Subresource Integrity (SRI) ensures the fetched file hasn't been tampered with. Required when using CDN-hosted scripts securely.

42. Document outline and heading hierarchy

<body>
  <h1>Site Name</h1>
  <main>
    <h2>Main Article Title</h2>
    <section>
      <h3>Section One</h3>
      <p>...</p>
      <h4>Subsection</h4>
    </section>
    <section>
      <h3>Section Two</h3>
    </section>
  </main>
</body>

Use headings in logical order (h1 → h2 → h3). Never skip levels. One h1 per page is best practice.

43. document.readyState and DOMContentLoaded

<script>
  // Fires when HTML is parsed (before images, CSS finish loading)
  document.addEventListener("DOMContentLoaded", () => {
    console.log("DOM ready");
  });

  // Fires when everything (images, CSS, fonts) is loaded
  window.addEventListener("load", () => {
    console.log("Page fully loaded");
  });
</script>

Understanding load order is critical for scripts that manipulate the DOM.

44. HTML document for a web app (SPA shell)

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My App</title>
  <link rel="stylesheet" href="/dist/styles.css">
</head>
<body>
  <div id="app"></div>
  <noscript>You need JavaScript to run this app.</noscript>
  <script type="module" src="/dist/main.js"></script>
</body>
</html>

Single-page applications often have a minimal HTML shell. type="module" scripts are deferred automatically.

45. XML declaration (not needed in HTML5)

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html>
<html xmlns="http://www.w3.org/1999/xhtml" lang="en">

XHTML required an XML declaration and namespace. Pure HTML5 does not. Avoid this pattern in new projects.

46. Tracking load performance with PerformanceNavigationTiming

<script>
  window.addEventListener("load", () => {
    const [nav] = performance.getEntriesByType("navigation");
    console.log("TTFB:", nav.responseStart - nav.requestStart, "ms");
    console.log("DOM loaded:", nav.domContentLoadedEventEnd - nav.startTime, "ms");
    console.log("Full load:", nav.loadEventEnd - nav.startTime, "ms");
  });
</script>

Use the Performance API to measure real user timing without external libraries.

47. Feature detection in head

<script>
  // Add classes based on feature support
  if ('IntersectionObserver' in window) {
    document.documentElement.classList.add('has-intersection-observer');
  }
  if (CSS.supports('display', 'grid')) {
    document.documentElement.classList.add('has-css-grid');
  }
</script>

Feature detection (not browser detection) applied to the root element lets CSS conditionally apply fallbacks.

48. Print stylesheet considerations

<link rel="stylesheet" href="print.css" media="print">

<!-- Or inline with @media print -->
<style>
  @media print {
    nav, aside, footer { display: none; }
    body { font-size: 12pt; color: black; }
    a::after { content: " (" attr(href) ")"; }
  }
</style>

Print styles hide navigation, show link URLs, and optimize for paper. Often forgotten.

49. Meta referrer policy

<meta name="referrer" content="strict-origin-when-cross-origin">

Controls how much referrer information is sent when the user follows links. strict-origin-when-cross-origin is the browser default since ~2020.

50. Full document template with all best practices

<!DOCTYPE html>
<html lang="en" dir="ltr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title — Brand Name</title>
  <meta name="description" content="150 char description for SEO.">
  <meta name="robots" content="index, follow">
  <link rel="canonical" href="https://example.com/page/">
  <link rel="icon" href="/favicon.svg" type="image/svg+xml">
  <link rel="manifest" href="/manifest.json">
  <meta name="theme-color" content="#ffffff">
  <!-- Open Graph -->
  <meta property="og:title" content="Page Title">
  <meta property="og:description" content="OG description.">
  <meta property="og:image" content="https://example.com/og.jpg">
  <meta property="og:url" content="https://example.com/page/">
  <meta property="og:type" content="website">
  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Page Title">
  <meta name="twitter:image" content="https://example.com/twitter.jpg">
  <!-- Performance hints -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preload" href="/hero.jpg" as="image">
  <link rel="stylesheet" href="/styles/main.css">
  <script src="/scripts/app.js" defer></script>
</head>
<body>
  <!-- page content -->
</body>
</html>

The gold-standard document template combining SEO, performance, accessibility, and social sharing.

Examples 4.1 — Audio and Video

BASIC (1–13)

1. Minimal audio element

<audio src="podcast.mp3" controls></audio>

The src attribute directly on the element. Simple but limits fallback options.

2. Audio with source child

<audio controls>
  <source src="podcast.mp3" type="audio/mpeg">
  Your browser does not support audio.
</audio>

Using <source> child allows multiple formats and a text fallback.

3. Minimal video element

<video src="demo.mp4" controls width="640"></video>

Direct src on the video element with controls and a width.

4. Video with source child

<video controls width="640">
  <source src="demo.mp4" type="video/mp4">
  Your browser does not support video.
</video>

Source child with MIME type helps browsers decide if they can play the file.

5. controls attribute

<audio controls>...</audio>
<video controls>...</video>

Shows the browser's built-in play/pause, volume, and seek controls.

6. autoplay attribute

<video src="intro.mp4" autoplay muted></video>

autoplay starts playback immediately. Most browsers require muted for autoplay to work.

7. loop attribute

<audio src="bg-music.mp3" loop muted></audio>

loop restarts playback from the beginning when it ends.

8. muted attribute

<video src="banner.mp4" autoplay loop muted></video>

muted starts playback silenced. Required for autoplay in most browsers.

9. preload attribute

<audio preload="none">...</audio>
<audio preload="metadata">...</audio>
<audio preload="auto">...</audio>

none: don't preload. metadata: only duration/dimensions. auto: browser decides (default).

10. poster attribute on video

<video controls poster="thumbnail.jpg">
  <source src="video.mp4" type="video/mp4">
</video>

poster shows an image before the video plays or while it loads.

11. width and height on video

<video controls width="1280" height="720">
  <source src="video.mp4" type="video/mp4">
</video>

Prevents layout shift by reserving space before the video loads.

12. Fallback text

<audio controls>
  <source src="sound.mp3" type="audio/mpeg">
  <p>Your browser doesn't support audio. <a href="sound.mp3">Download the file</a>.</p>
</audio>

Content between tags is shown in browsers that don't support the element.

13. audio/mpeg MIME type

<source src="file.mp3" type="audio/mpeg">
<source src="file.ogg" type="audio/ogg">

Common audio MIME types. The browser picks the first it can play.

INTERMEDIATE (14–26)

14. Multiple source formats for audio

<audio controls>
  <source src="podcast.mp3" type="audio/mpeg">
  <source src="podcast.ogg" type="audio/ogg">
  <source src="podcast.wav" type="audio/wav">
  Your browser does not support audio.
</audio>

Browser tries each source in order; plays the first supported format.

15. Multiple source formats for video

<video controls width="640">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <source src="video.ogv" type="video/ogg">
</video>

mp4 (H.264) has the widest support; webm is smaller but needs newer browsers.

16. track element for subtitles

<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="subtitles" src="subtitles/en.vtt" srclang="en" label="English">
  <track kind="subtitles" src="subtitles/fr.vtt" srclang="fr" label="Français">
</video>

track provides text tracks. kind can be: subtitles, captions, descriptions, chapters, metadata.

17. track with default attribute

<track kind="subtitles" src="en.vtt" srclang="en" label="English" default>

default enables this track automatically when the user hasn't selected one.

18. WebVTT cue format

WEBVTT

00:00:01.000 --> 00:00:04.000
Hello and welcome to this video.

00:00:05.500 --> 00:00:08.000
Today we'll cover HTML media elements.

WebVTT format for subtitle/caption files (.vtt). Timestamps are HH:MM:SS.mmm.

19. Controlling media with JavaScript

const video = document.getElementById('my-video');
video.play();
video.pause();
video.currentTime = 30; // seek to 30 seconds
video.volume = 0.5;     // 50% volume
video.playbackRate = 1.5; // 1.5x speed

HTMLMediaElement API for programmatic control.

20. Media events

video.addEventListener('play', () => console.log('playing'));
video.addEventListener('pause', () => console.log('paused'));
video.addEventListener('ended', () => console.log('ended'));
video.addEventListener('timeupdate', () => {
  console.log(video.currentTime);
});

Key events: play, pause, ended, timeupdate, loadedmetadata, canplay, error.

21. canplay and canplaythrough events

video.addEventListener('canplay', () => {
  console.log('Can start playing');
});
video.addEventListener('canplaythrough', () => {
  console.log('Can play without buffering');
});

canplay fires when enough data is available to start; canplaythrough when the whole video can play without pausing.

22. Checking readyState

// readyState values:
// 0 HAVE_NOTHING
// 1 HAVE_METADATA
// 2 HAVE_CURRENT_DATA
// 3 HAVE_FUTURE_DATA
// 4 HAVE_ENOUGH_DATA
if (video.readyState >= 3) {
  video.play();
}

readyState tells you how much data is loaded.

23. Custom video controls

<video id="vid" src="movie.mp4"></video>
<button onclick="document.getElementById('vid').play()">Play</button>
<button onclick="document.getElementById('vid').pause()">Pause</button>
<input type="range" min="0" max="1" step="0.1" value="1"
  oninput="document.getElementById('vid').volume = this.value">

Omit controls attribute and build custom UI with JS for full design control.

24. Fullscreen API

const btn = document.getElementById('fullscreen-btn');
btn.addEventListener('click', () => {
  if (video.requestFullscreen) {
    video.requestFullscreen();
  }
});

requestFullscreen() makes the video go fullscreen. Also available on any element.

25. Picture-in-Picture

video.addEventListener('click', async () => {
  if (document.pictureInPictureElement) {
    await document.exitPictureInPicture();
  } else {
    await video.requestPictureInPicture();
  }
});

Picture-in-Picture API lets videos float in a small overlay window.

26. disablepictureinpicture attribute

<video src="video.mp4" controls disablepictureinpicture
       controlslist="nodownload noremoteplayback"></video>

Restricts browser controls. controlslist values: nodownload, nofullscreen, noremoteplayback.

NESTED (27–38)

27. Video inside a figure

<figure>
  <video controls width="640">
    <source src="demo.mp4" type="video/mp4">
  </video>
  <figcaption>Demo of the new feature — recorded 2026-01</figcaption>
</figure>

Wrapping in figure+figcaption provides a semantic caption.

28. Audio player UI

<div class="player">
  <audio id="player" src="episode.mp3" preload="metadata"></audio>
  <button id="play-btn" aria-label="Play">▶</button>
  <span id="time">0:00</span>
  <input type="range" id="seek" value="0">
</div>

Custom audio player with accessible play button and seek bar.

29. Video with multiple tracks and fallback

<video controls width="800" poster="thumb.jpg">
  <source src="film.mp4" type="video/mp4">
  <source src="film.webm" type="video/webm">
  <track kind="subtitles" src="en.vtt" srclang="en" label="English" default>
  <track kind="subtitles" src="es.vtt" srclang="es" label="Español">
  <track kind="chapters" src="chapters.vtt" srclang="en" label="Chapters">
  <p>Download: <a href="film.mp4">MP4</a> | <a href="film.webm">WebM</a></p>
</video>

Production-ready video with multiple formats, subtitles, and download fallback.

30. Responsive video with aspect ratio

<style>
.video-wrap { position: relative; padding-bottom: 56.25%; height: 0; }
.video-wrap video { position: absolute; top:0; left:0; width:100%; height:100%; }
</style>
<div class="video-wrap">
  <video controls>
    <source src="video.mp4" type="video/mp4">
  </video>
</div>

Classic padding-bottom trick for 16:9 responsive video (56.25% = 9/16).

31. aspect-ratio CSS approach

<style>
video { width: 100%; aspect-ratio: 16 / 9; }
</style>
<video controls>
  <source src="video.mp4" type="video/mp4">
</video>

Modern CSS aspect-ratio is simpler than the padding-bottom hack.

32. MediaSource Extensions

const ms = new MediaSource();
video.src = URL.createObjectURL(ms);
ms.addEventListener('sourceopen', () => {
  const sb = ms.addSourceBuffer('video/mp4; codecs="avc1.42E01E"');
  fetch('segment.mp4').then(r => r.arrayBuffer()).then(data => {
    sb.appendBuffer(data);
  });
});

MSE enables adaptive streaming (HLS/DASH) by feeding segments to a SourceBuffer.

33. Progress bar with timeupdate

<video id="v" src="video.mp4" controls></video>
<progress id="bar" value="0" max="1"></progress>
<script>
v.addEventListener('timeupdate', () => {
  bar.value = v.currentTime / v.duration;
});
</script>

Custom progress bar synchronized with video playback position.

34. Detecting codec support

const v = document.createElement('video');
const canH264 = v.canPlayType('video/mp4; codecs="avc1.42E01E"');
const canVP9  = v.canPlayType('video/webm; codecs="vp9"');
// Returns: "" (no), "maybe", or "probably"
console.log(canH264, canVP9);

canPlayType() returns "" (unsupported), "maybe", or "probably" for codec support.

35. Capturing a frame with canvas

const canvas = document.createElement('canvas');
canvas.width  = video.videoWidth;
canvas.height = video.videoHeight;
canvas.getContext('2d').drawImage(video, 0, 0);
const dataURL = canvas.toDataURL('image/png');

Draw the current video frame to canvas to capture a screenshot.

36. Media session API

navigator.mediaSession.metadata = new MediaMetadata({
  title: 'Episode 42',
  artist: 'My Podcast',
  album: 'Season 3',
  artwork: [{ src: 'cover.jpg', sizes: '512x512', type: 'image/jpeg' }]
});
navigator.mediaSession.setActionHandler('play', () => audio.play());
navigator.mediaSession.setActionHandler('pause', () => audio.pause());

Media Session API populates OS media notification with title, artist, and controls.

37. Preloading metadata only

<video id="v" src="big-file.mp4" preload="metadata"></video>
<script>
v.addEventListener('loadedmetadata', () => {
  console.log('Duration:', v.duration);
  console.log('Dimensions:', v.videoWidth, 'x', v.videoHeight);
});
</script>

preload="metadata" downloads just enough to know duration and dimensions.

38. Intersection Observer for lazy video

const observer = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.src = e.target.dataset.src;
      observer.unobserve(e.target);
    }
  });
});
document.querySelectorAll('video[data-src]').forEach(v => observer.observe(v));

Lazy-load videos by setting data-src instead of src, then swapping with IntersectionObserver.

ADVANCED (39–50)

39. Web Audio API basic setup

const ctx = new AudioContext();
const source = ctx.createMediaElementSource(audioEl);
const gainNode = ctx.createGain();
source.connect(gainNode).connect(ctx.destination);
gainNode.gain.value = 0.5; // 50% volume

Web Audio API enables advanced audio processing: EQ, reverb, visualizations, etc.

40. Visualizing audio with analyser

const analyser = ctx.createAnalyser();
source.connect(analyser).connect(ctx.destination);
const data = new Uint8Array(analyser.frequencyBinCount);
function draw() {
  analyser.getByteFrequencyData(data);
  // draw data to canvas...
  requestAnimationFrame(draw);
}
draw();

AnalyserNode provides frequency/time-domain data for audio visualizations.

41. getUserMedia for camera

const stream = await navigator.mediaDevices.getUserMedia({ video: true, audio: true });
videoEl.srcObject = stream;

getUserMedia captures camera/microphone. Always requires user permission.

42. Screen capture

const stream = await navigator.mediaDevices.getDisplayMedia({ video: true });
videoEl.srcObject = stream;

getDisplayMedia captures the screen or a window/tab.

43. MediaRecorder API

const recorder = new MediaRecorder(stream);
const chunks = [];
recorder.ondataavailable = e => chunks.push(e.data);
recorder.onstop = () => {
  const blob = new Blob(chunks, { type: 'video/webm' });
  videoEl.src = URL.createObjectURL(blob);
};
recorder.start();
setTimeout(() => recorder.stop(), 5000);

MediaRecorder records a MediaStream to a downloadable blob.

44. Remote playback API

const availability = await video.remote.watchAvailability(available => {
  castBtn.disabled = !available;
});
castBtn.addEventListener('click', () => video.remote.prompt());

Remote Playback API enables casting to Chromecast, AirPlay, etc.

45. Encrypted Media Extensions (EME)

video.addEventListener('encrypted', async e => {
  const session = mediaKeys.createSession();
  await session.generateRequest(e.initDataType, e.initData);
});
const access = await navigator.requestMediaKeySystemAccess('com.widevine.alpha', config);
const mediaKeys = await access.createMediaKeys();
await video.setMediaKeys(mediaKeys);

EME enables DRM-protected content (Widevine, PlayReady, FairPlay).

46. Adaptive Bitrate with hls.js

<script src="https://cdn.jsdelivr.net/npm/hls.js"></script>
<script>
if (Hls.isSupported()) {
  const hls = new Hls();
  hls.loadSource('stream.m3u8');
  hls.attachMedia(video);
} else if (video.canPlayType('application/vnd.apple.mpegurl')) {
  video.src = 'stream.m3u8'; // Safari native HLS
}
</script>

hls.js brings HLS adaptive streaming to browsers without native support.

47. Video text track programmatic access

const tracks = video.textTracks;
for (const track of tracks) {
  track.mode = 'showing'; // or 'hidden' or 'disabled'
  track.addEventListener('cuechange', () => {
    const cues = track.activeCues;
    console.log(cues[0]?.text);
  });
}

Access and control text tracks and active cues programmatically.

49. Spatial audio with PannerNode

const panner = ctx.createPanner();

50. AV1 codec with WebM

<video controls>
  <source src="video.av1.mp4" type='video/mp4; codecs="av01.0.08M.08"'>
  <source src="video.vp9.webm" type='video/webm; codecs="vp9"'>
  <source src="video.h264.mp4" type='video/mp4; codecs="avc1.42E01E"'>
</video>

AV1 is the most efficient codec; include VP9 and H.264 fallbacks for older browsers.

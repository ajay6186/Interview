Examples 4.2 — Responsive Images

BASIC (1–13)

1. Basic img element

<img src="photo.jpg" alt="A mountain landscape">

Always include alt text. Empty alt="" for decorative images, descriptive text for informational ones.

2. width and height attributes

<img src="photo.jpg" alt="Mountain" width="800" height="600">

Always set width and height to prevent layout shift (CLS). Browser reserves space before image loads.

3. loading="lazy"

<img src="below-fold.jpg" alt="Below the fold" loading="lazy">

Defers loading until the image approaches the viewport. Never use on above-the-fold images.

4. loading="eager" (default)

<img src="hero.jpg" alt="Hero" loading="eager">

Loads immediately. Default behavior. Use explicitly on important above-the-fold images.

5. decoding="async"

<img src="photo.jpg" alt="Photo" loading="lazy" decoding="async">

Allows browser to decode the image off the main thread, preventing jank.

6. srcset with width descriptors

<img src="hero-800.jpg" alt="Hero"
     srcset="hero-400.jpg 400w, hero-800.jpg 800w, hero-1600.jpg 1600w">

Browser picks the best size based on device width and pixel density. src is the fallback.

7. srcset with pixel density descriptors

<img src="logo.png" alt="Logo"
     srcset="logo.png 1x, logo@2x.png 2x, logo@3x.png 3x">

1x/2x/3x descriptors serve high-DPI versions to retina screens.

8. sizes attribute

<img src="photo.jpg" alt="Photo"
     srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1200w"
     sizes="(max-width: 600px) 400px, (max-width: 900px) 800px, 1200px">

sizes tells the browser how wide the image will be at each breakpoint, enabling optimal selection.

9. picture element

<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Description">
</picture>

picture allows format fallbacks. Browser uses first supported source; img is always the fallback.

10. picture for art direction

<picture>
  <source media="(max-width: 600px)" srcset="portrait.jpg">
  <source media="(min-width: 601px)" srcset="landscape.jpg">
  <img src="landscape.jpg" alt="Photo">
</picture>

media attribute selects different images for different viewport sizes — not just different resolutions.

11. Decorative image

<img src="divider.png" alt="">

Empty alt hides the image from screen readers. Use for purely decorative images.

12. fetchpriority

<img src="hero.jpg" alt="Hero" fetchpriority="high">
<img src="thumbnail.jpg" alt="Thumb" fetchpriority="low">

fetchpriority hints the browser about image importance. Use "high" for LCP images.

13. crossorigin attribute

<img src="https://cdn.example.com/photo.jpg" alt="CDN photo"
     crossorigin="anonymous">

Required for images drawn to canvas from a different origin without tainting it.

INTERMEDIATE (14–26)

14. Complete responsive hero image

<img src="hero-1200.jpg"
     alt="Mountain landscape at sunset"
     width="1200" height="600"
     srcset="hero-400.jpg 400w,
             hero-800.jpg 800w,
             hero-1200.jpg 1200w,
             hero-2400.jpg 2400w"
     sizes="100vw"
     fetchpriority="high"
     decoding="async">

Full-width hero best practice: srcset for resolution, sizes="100vw", high fetch priority.

15. Combining picture + srcset + format

<picture>
  <source type="image/avif"
    srcset="photo-400.avif 400w, photo-800.avif 800w"
    sizes="(max-width: 600px) 400px, 800px">
  <source type="image/webp"
    srcset="photo-400.webp 400w, photo-800.webp 800w"
    sizes="(max-width: 600px) 400px, 800px">
  <img src="photo-800.jpg" alt="Photo" width="800" height="600">
</picture>

Best of both worlds: format negotiation (avif→webp→jpg) plus resolution switching.

16. Image in a grid layout

<style>
.grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 16px; }
.grid img { width: 100%; height: 200px; object-fit: cover; }
</style>
<div class="grid">
  <img src="a.jpg" alt="A" loading="lazy">
  <img src="b.jpg" alt="B" loading="lazy">
  <img src="c.jpg" alt="C" loading="lazy">
</div>

object-fit: cover fills the fixed area without distortion. Use loading="lazy" for grid images.

17. sizes with calc()

<img src="card.jpg" alt="Card image"
     srcset="card-300.jpg 300w, card-600.jpg 600w, card-900.jpg 900w"
     sizes="(max-width: 800px) calc(50vw - 32px), 400px">

calc() in sizes accounts for grid gaps, padding, and margins in layout calculations.

18. WebP with JPEG fallback

<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Product photo" width="400" height="400">
</picture>

WebP is ~30% smaller than JPEG. All modern browsers support it; IE11 gets the JPEG fallback.

19. AVIF with WebP and JPEG fallbacks

<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Photo" width="800" height="600">
</picture>

AVIF is ~50% smaller than JPEG. Add WebP as intermediate fallback for broader browser support.

20. SVG as img src

<img src="logo.svg" alt="Company logo" width="200" height="80">

SVG scales to any resolution. Always set intrinsic dimensions for layout stability.

21. Responsive SVG with picture

<picture>
  <source media="(max-width: 480px)" srcset="logo-small.svg">
  <img src="logo-full.svg" alt="Logo" width="300" height="100">
</picture>

Different SVG variations for mobile (simplified) vs desktop (detailed).

22. CSS background-image vs img

<!-- Decorative background -->
<div style="background-image: url('pattern.jpg')">Content</div>

<!-- Content image -->
<img src="product.jpg" alt="Blue sneakers">

background-image is for decoration (no alt, not indexed). img is for content images (accessible, SEO).

23. onerror fallback

<img src="avatar.jpg" alt="User avatar"
     onerror="this.src='default-avatar.png'; this.onerror=null;">

onerror switches to a fallback image when the original fails to load. null prevents infinite loop.

24. usemap for image maps

<img src="world-map.jpg" alt="World map" usemap="#continents">
<map name="continents">
  <area shape="rect" coords="100,50,200,150" href="europe.html" alt="Europe">
  <area shape="circle" coords="350,200,80" href="asia.html" alt="Asia">
</map>

Image maps define clickable areas. Shape options: rect, circle, poly.

25. intrinsicsize (not yet widely supported)

<!-- Modern: always specify width+height attributes -->
<img src="photo.jpg" alt="Photo" width="800" height="600">

Setting width+height attributes lets browsers calculate aspect ratio and reserve space without loading the image.

26. Image color-scheme

<picture>
  <source srcset="logo-dark.png" media="(prefers-color-scheme: dark)">
  <img src="logo-light.png" alt="Logo">
</picture>

Serve different images for dark and light mode using media queries in picture.

NESTED (27–38)

27. Figure with responsive image

<figure>
  <picture>
    <source srcset="chart.avif" type="image/avif">
    <img src="chart.png" alt="Q4 revenue chart showing 20% growth"
         width="800" height="400" loading="lazy">
  </picture>
  <figcaption>Q4 2025 Revenue — 20% year-over-year growth</figcaption>
</figure>

Nested picture inside figure with descriptive figcaption — semantic and accessible.

28. Card component with responsive image

<article class="card">
  <picture>
    <source srcset="thumb.webp" type="image/webp">
    <img src="thumb.jpg" alt="" width="400" height="200"
         loading="lazy" class="card__img">
  </picture>
  <div class="card__body">
    <h2>Article Title</h2>
    <p>Summary text...</p>
  </div>
</article>

Card pattern: empty alt (decorative), lazy loading, webp format, fixed dimensions.

29. Avatar with srcset

<img src="avatar-48.jpg" alt="Jane Doe's avatar"
     srcset="avatar-48.jpg 1x, avatar-96.jpg 2x, avatar-144.jpg 3x"
     width="48" height="48"
     style="border-radius: 50%">

Pixel density srcset for UI elements at fixed CSS sizes (avatars, icons, logos).

30. Lazy loading below-fold gallery

<div class="gallery">
  <img src="p1.jpg" alt="Photo 1" width="300" height="200" loading="lazy" decoding="async">
  <img src="p2.jpg" alt="Photo 2" width="300" height="200" loading="lazy" decoding="async">
  <img src="p3.jpg" alt="Photo 3" width="300" height="200" loading="lazy" decoding="async">
</div>

Always lazy-load gallery images. Always specify dimensions to prevent layout shift.

31. Responsive product images

<picture>
  <source media="(max-width: 480px)"
    srcset="product-mobile.avif" type="image/avif">
  <source media="(max-width: 480px)"
    srcset="product-mobile.webp" type="image/webp">
  <source srcset="product-desktop.avif" type="image/avif">
  <source srcset="product-desktop.webp" type="image/webp">
  <img src="product-desktop.jpg" alt="Red leather wallet"
       width="600" height="600">
</picture>

Full art-direction + format negotiation for e-commerce product images.

32. Blurred placeholder technique

<div style="position:relative;background:url(tiny-blur.jpg) center/cover">
  <img src="full.jpg" alt="Full image"
       loading="lazy" style="opacity:0;transition:opacity 0.3s"
       onload="this.style.opacity=1">
</div>

Show a tiny blurred placeholder (LQIP) while the full image loads, then fade it in.

33. IntersectionObserver lazy loading (JS)

<img data-src="photo.jpg" alt="Lazy photo" class="lazy">
<script>
const obs = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.src = e.target.dataset.src;
      obs.unobserve(e.target);
    }
  });
});
document.querySelectorAll('img.lazy').forEach(img => obs.observe(img));
</script>

Custom lazy loading with IntersectionObserver — useful for older browser support.

34. Object-fit in responsive layout

<style>
.hero { width: 100%; height: 50vh; }
.hero img { width: 100%; height: 100%; object-fit: cover; object-position: center top; }
</style>
<div class="hero">
  <img src="hero.jpg" alt="Team photo">
</div>

object-fit: cover fills fixed area; object-position controls focal point of the crop.

35. srcset with sizes for card grid

<img src="card.jpg" alt="Card"
     srcset="card-300.jpg 300w, card-600.jpg 600w, card-900.jpg 900w"
     sizes="(max-width: 480px) 100vw,
            (max-width: 900px) calc(50vw - 32px),
            300px">

Sizes for a 1→2→3 column responsive grid. 32px accounts for gap/padding.

36. picture with orientation

<picture>
  <source media="(orientation: portrait)"  srcset="portrait.jpg">
  <source media="(orientation: landscape)" srcset="landscape.jpg">
  <img src="landscape.jpg" alt="Photo">
</picture>

Serve portrait or landscape crops based on device orientation.

37. Sprite sheet

<style>
.icon { width:32px; height:32px; background: url('sprite.png') no-repeat; }
.icon-home  { background-position: 0 0; }
.icon-user  { background-position: -32px 0; }
.icon-cart  { background-position: -64px 0; }
</style>
<span class="icon icon-home" role="img" aria-label="Home"></span>

CSS sprite sheet reduces HTTP requests. Add role+aria-label for accessibility.

38. Base64 inline image

<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mNk+M9QDwADhgGAWjR9awAAAABJRU5ErkJggg=="
     alt="1px red dot" width="1" height="1">

Inline base64 avoids an HTTP request. Only use for tiny images — base64 is 33% larger than binary.

ADVANCED (39–50)

39. preload hint in head

<link rel="preload" as="image" href="hero.webp" type="image/webp">
<link rel="preload" as="image" href="hero.jpg">

Preloads the LCP image early, before the browser parses the img tag. Critical for performance.

40. preload with imagesrcset

<link rel="preload" as="image"
      imagesrcset="hero-400.jpg 400w, hero-800.jpg 800w"
      imagesizes="(max-width: 600px) 400px, 800px"
      fetchpriority="high">

imagesrcset/imagesizes in link[rel=preload] matches the img srcset/sizes for optimal preloading.

41. Content negotiation via Accept header

<!-- Server-side: respond with AVIF if Accept: image/avif -->
<!-- Client-side: just use a picture element -->
<picture>
  <source srcset="/image?format=avif" type="image/avif">
  <img src="/image?format=jpg" alt="Server-negotiated format">
</picture>

Combine server-side format negotiation with picture element for maximum compatibility.

42. Preventing image dragging

<img src="photo.jpg" alt="Photo" draggable="false"
     style="-webkit-user-drag: none; user-select: none;">

draggable="false" prevents default drag behavior. Add CSS for cross-browser coverage.

43. Canvas-based image processing

const img = new Image();
img.crossOrigin = 'anonymous';
img.onload = () => {
  const ctx = canvas.getContext('2d');
  ctx.drawImage(img, 0, 0);
  const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
  // process pixels...
};
img.src = 'photo.jpg';

Draw images to canvas to access pixel data for filters, analysis, or manipulation.

44. PerformanceObserver for LCP

new PerformanceObserver(list => {
  const entries = list.getEntries();
  const last = entries[entries.length - 1];
  console.log('LCP element:', last.element);
  console.log('LCP time:', last.startTime);
}).observe({ type: 'largest-contentful-paint', buffered: true });

Monitor which image is your LCP (Largest Contentful Paint) element.

45. Detecting WebP support

async function supportsWebP() {
  const webP = new Image();
  webP.src = 'data:image/webp;base64,UklGRjoAAABXRUJQVlA4IC4AAACyAgCdASoCAAIALmk0mk0iIiIiIgBoSygABc6WWgAA/veff/0PP8bA//LwYAAA';
  return new Promise(r => webP.onload = webP.onerror = () => r(webP.height === 2));

Feature-detect WebP support. Modern approach: just use picture element instead.

46. CSS image-rendering for pixel art

<img src="sprite.png" alt="Pixel art"
     width="320" height="320"
     style="image-rendering: pixelated">

image-rendering: pixelated preserves crisp edges when scaling up pixel art.

47. Service Worker image caching

self.addEventListener('fetch', e => {
  if (e.request.destination === 'image') {
    e.respondWith(
      caches.match(e.request).then(r => r || fetch(e.request))
    );
  }
});

Cache-first strategy for images: serve from cache if available, else fetch and cache.

48. Blur-up progressive loading

<style>
.progressive { filter: blur(20px); transition: filter 0.5s; }
.progressive.loaded { filter: none; }
</style>
<img src="tiny.jpg" data-src="full.jpg" alt="Progressive" class="progressive">
<script>
document.querySelectorAll('.progressive').forEach(img => {
  const full = new Image();
  full.onload = () => { img.src = full.src; img.classList.add('loaded'); };
  full.src = img.dataset.src;
});
</script>

Start with tiny blurred image, load full size in background, fade out blur on load.

49. Resolution Media Query

<picture>
  <source media="(min-resolution: 2dppx)" srcset="photo@2x.jpg">
  <img src="photo.jpg" alt="High-res photo">
</picture>

min-resolution targets high-DPI screens. Equivalent to srcset 2x but in picture/source.

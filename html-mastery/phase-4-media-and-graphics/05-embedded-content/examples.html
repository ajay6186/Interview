Examples 4.5 — Embedded Content

BASIC (1–13)

1. Minimal iframe

<iframe src="https://example.com" width="600" height="400" title="Example website"></iframe>

Always include title for accessibility. Always set width and height.

2. title attribute (required for accessibility)

<iframe src="map.html" title="Interactive map of store locations"></iframe>

The title attribute describes the iframe's content to screen readers. Never omit it.

3. sandbox attribute

<iframe src="widget.html" sandbox></iframe>

Empty sandbox blocks scripts, forms, popups, same-origin access, and more. The safest default.

4. sandbox with permissions

<iframe src="widget.html" sandbox="allow-scripts allow-same-origin"></iframe>

Re-grant specific permissions. Values: allow-scripts, allow-same-origin, allow-forms, allow-popups, allow-top-navigation.

5. loading="lazy" for iframes

<iframe src="comments.html" loading="lazy" title="Comments section"
        width="100%" height="500"></iframe>

Defers loading until the iframe approaches the viewport. Great for below-fold content.

6. allow attribute (Feature Policy)

<iframe src="video-player.html"
        allow="fullscreen; autoplay; encrypted-media"
        title="Video player"></iframe>

allow controls which browser features (camera, fullscreen, payment, etc.) the iframe can use.

7. name attribute for frame targeting

<iframe name="content-frame" src="page.html" title="Content"></iframe>
<a href="other-page.html" target="content-frame">Load other page</a>

Named iframes can be targeted by links and forms using the target attribute.

8. srcdoc attribute

<iframe srcdoc="<p>Hello <strong>world</strong>!</p>"
        title="Inline content"></iframe>

srcdoc embeds HTML directly without a separate URL request. Automatically sandboxed.

9. referrerpolicy

<iframe src="https://partner.com/widget"
        referrerpolicy="no-referrer"
        title="Partner widget"></iframe>

Controls what referrer info is sent. no-referrer, origin, strict-origin-when-cross-origin, etc.

10. Blocking navigation with sandbox

<iframe src="user-content.html"
        sandbox="allow-scripts"
        title="User content"></iframe>

Without allow-top-navigation, the iframe cannot redirect the parent page.

11. iframe with style

<iframe src="sidebar.html"
        title="Sidebar"
        style="border: none; width: 300px; height: 100vh;"></iframe>

Use border: none to remove the default iframe border. Control dimensions with CSS.

12. Embedding a YouTube video

<iframe width="560" height="315"
        src="https://www.youtube.com/embed/VIDEO_ID"
        title="YouTube video player"
        frameborder="0"
        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
        allowfullscreen></iframe>

YouTube's standard embed code. frameborder="0" is deprecated; use CSS border: none instead.

13. about:blank iframe

<iframe id="sandbox" src="about:blank" sandbox title="Sandbox"></iframe>
<script>
const frame = document.getElementById('sandbox');
frame.srcdoc = '<p>Dynamically injected content</p>';
</script>

about:blank loads an empty page. Use srcdoc to inject content dynamically.

INTERMEDIATE (14–26)

14. postMessage communication

// Parent → iframe:
iframe.contentWindow.postMessage({ action: 'setTheme', value: 'dark' }, 'https://widget.com');

// Inside iframe:
window.addEventListener('message', e => {
  if (e.origin !== 'https://parent.com') return; // always verify origin
  console.log(e.data);
});

postMessage is the secure way to communicate cross-origin. Always verify e.origin.

15. Receiving messages from iframe

// In parent:
window.addEventListener('message', e => {
  if (e.origin !== 'https://widget.com') return;
  if (e.data.type === 'resize') {
    iframe.height = e.data.height + 'px';
  }
});

// In iframe (to auto-resize parent):
parent.postMessage({ type: 'resize', height: document.body.scrollHeight }, '*');

Auto-resizing iframe: iframe posts its height to parent, parent adjusts the iframe element.

16. Embedding Google Maps

<iframe
  src="https://www.google.com/maps/embed?pb=..."
  width="600" height="450"
  style="border:0"
  allowfullscreen
  loading="lazy"
  referrerpolicy="no-referrer-when-downgrade"
  title="Store location map"></iframe>

Google Maps embed pattern. loading="lazy" defers it until near viewport.

17. Responsive iframe wrapper

<style>
.iframe-wrap { position: relative; padding-bottom: 56.25%; height: 0; overflow: hidden; }
.iframe-wrap iframe { position: absolute; top: 0; left: 0; width: 100%; height: 100%; border: 0; }
</style>
<div class="iframe-wrap">
  <iframe src="video.html" title="Video"></iframe>
</div>

Same 16:9 padding-bottom trick as for video. Makes iframe aspect-ratio responsive.

18. Embedding a PDF

<iframe src="document.pdf" width="100%" height="600px"
        title="Annual report PDF">
  <p>Your browser doesn't support PDFs.
     <a href="document.pdf">Download the PDF</a>.</p>
</iframe>

Browsers render PDFs inline. Provide a download fallback for browsers that don't support it.

19. Content Security Policy for iframes

<!-- In HTTP response headers: -->
Content-Security-Policy: frame-ancestors 'self' https://partner.com

<!-- Or in meta tag (limited): -->
<meta http-equiv="Content-Security-Policy"
      content="frame-src 'self' https://youtube.com">

frame-ancestors restricts who can embed your page. frame-src restricts what your page can embed.

20. X-Frame-Options header

<!-- HTTP response header: -->
X-Frame-Options: DENY           <!-- never embeddable -->
X-Frame-Options: SAMEORIGIN     <!-- only same origin -->
X-Frame-Options: ALLOW-FROM https://partner.com  <!-- deprecated -->

X-Frame-Options prevents clickjacking by controlling iframe embedding. Use CSP frame-ancestors instead.

21. Intersection Observer to lazy-load iframe

<iframe data-src="heavy-widget.html" title="Widget" class="lazy-frame"></iframe>
<script>
const obs = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.src = e.target.dataset.src;
      obs.unobserve(e.target);
    }
  });
});
document.querySelectorAll('.lazy-frame').forEach(f => obs.observe(f));
</script>

Native loading="lazy" has limited support; IntersectionObserver provides a reliable polyfill.

22. iframe security checklist

<!-- Secure iframe pattern: -->
<iframe
  src="https://trusted-vendor.com/widget"
  sandbox="allow-scripts allow-same-origin allow-forms"
  allow="payment"
  referrerpolicy="strict-origin-when-cross-origin"
  loading="lazy"
  title="Payment widget"
  width="400" height="300"></iframe>

Security checklist: sandbox, specific allow, strict referrer, title for a11y, lazy loading.

23. Detecting iframe load

const iframe = document.getElementById('my-frame');
iframe.addEventListener('load', () => {
  console.log('iframe loaded:', iframe.src);
  // Can now access same-origin iframe:
  // iframe.contentDocument, iframe.contentWindow
});
iframe.addEventListener('error', () => {
  console.error('iframe failed to load');
});

Use load event to know when iframe content is ready. Cross-origin iframes limit DOM access.

24. Accessing same-origin iframe DOM

const frame = document.getElementById('my-frame');
frame.addEventListener('load', () => {
  const doc = frame.contentDocument;
  const heading = doc.querySelector('h1');
  heading.textContent = 'Modified by parent';
});
// or: frame.contentWindow.document

Same-origin iframes allow full DOM access. Cross-origin iframes block this for security.

25. object element (legacy)

<object data="presentation.pdf" type="application/pdf"
        width="600" height="400">
  <p>Alternative: <a href="presentation.pdf">Download PDF</a></p>
</object>

object embeds external resources. Prefer iframe for HTML content; use object for PDFs and legacy plugins.

26. embed element (legacy)

<embed src="animation.swf" type="application/x-shockwave-flash"
       width="550" height="400">

embed is void element with no fallback content. Flash is dead; use it for SVG or plugin content only.

NESTED (27–38)

27. Lazy-loaded YouTube with facade

<style>
.yt-facade { position:relative; cursor:pointer; background:#000; }
.yt-facade img { width:100%; opacity:0.8; }
.yt-facade::after { content:"▶"; position:absolute; top:50%; left:50%;
  transform:translate(-50%,-50%); font-size:60px; color:white; }
</style>
<div class="yt-facade" data-video-id="dQw4w9WgXcQ"
     onclick="this.innerHTML='<iframe src=\'https://www.youtube.com/embed/'+this.dataset.videoId+'?autoplay=1\' ...></iframe>'">
  <img src="https://img.youtube.com/vi/dQw4w9WgXcQ/maxresdefault.jpg" alt="Video thumbnail">
</div>

YouTube facade pattern: show thumbnail, load real iframe on click. Saves ~500KB per embed.

28. Codepen-style split editor

<div style="display:grid; grid-template-columns: 1fr 1fr; height:400px">
  <textarea id="code" style="height:100%; font-family:monospace"></textarea>
  <iframe id="preview" title="Preview" style="border:none; height:100%"></iframe>
</div>
<script>
document.getElementById('code').addEventListener('input', e => {
  document.getElementById('preview').srcdoc = e.target.value;
});
</script>

Live preview using srcdoc: update iframe content on every keystroke in the textarea.

29. iframe in a modal dialog

<dialog id="modal">
  <iframe id="modal-frame" src="" title="Modal content"
          width="600" height="400"></iframe>
  <button onclick="closeModal()">Close</button>
</dialog>
<script>
function openModal(url) {
  document.getElementById('modal-frame').src = url;
  document.getElementById('modal').showModal();
}
function closeModal() {
  document.getElementById('modal-frame').src = '';
  document.getElementById('modal').close();
}
</script>

Clear iframe src on close to stop media playback. Set src on open for clean load.

30. Sandboxed code execution

function runCode(code) {
  const iframe = document.createElement('iframe');
  iframe.sandbox = 'allow-scripts';
  iframe.srcdoc = `<script>
    try {
      const result = eval(${JSON.stringify(code)});
      parent.postMessage({ type: 'result', value: String(result) }, '*');
    } catch(e) {
      parent.postMessage({ type: 'error', value: e.message }, '*');
    }
  </script>`;
  document.body.appendChild(iframe);
  setTimeout(() => iframe.remove(), 5000);
}

Safe code execution: run untrusted code in a sandboxed iframe, receive results via postMessage.

31. iframe resize observer

// Inside the iframe:
const resizeObserver = new ResizeObserver(() => {
  parent.postMessage({
    type: 'resize',
    height: document.documentElement.scrollHeight
  }, '*');
});
resizeObserver.observe(document.documentElement);

// In parent:
window.addEventListener('message', e => {
  if (e.data.type === 'resize') {
    iframe.style.height = e.data.height + 'px';
  }
});

Auto-height iframe: observe content height changes and notify parent to resize.

32. Codesandbox-style iframe preview

const files = {
  'index.html': '<link rel="stylesheet" href="style.css"><h1>Hello</h1>',
  'style.css': 'h1 { color: coral; }'
};
// Serve files from a Service Worker or blob URLs
const htmlBlob = new Blob([files['index.html']], { type: 'text/html' });
iframe.src = URL.createObjectURL(htmlBlob);

Multiple file preview: create blob URLs for each file; link them from the HTML blob.

33. iframe with CSP header response

<!-- In the embedded page's HTTP response: -->
Content-Security-Policy: default-src 'self'; script-src 'none'; frame-ancestors https://parent.com

<!-- In the parent page: -->
<iframe src="https://secure-widget.com" title="Widget" sandbox="allow-same-origin"></iframe>

Double protection: the embedded page uses CSP to restrict itself, plus the parent sandboxes it.

34. iframe focus management

// Send focus into iframe:
iframe.contentWindow.focus();

// Detect when iframe gets focus (same-origin only):
iframe.addEventListener('load', () => {
  iframe.contentWindow.addEventListener('focus', () => {
    console.log('iframe focused');
  });
});

// Trap focus inside iframe for modals (custom logic needed)

Focus management between parent and iframe is tricky. Tab order continues into iframe naturally.

35. Embedding Stripe payment

<!-- Stripe handles iframe creation automatically -->
<div id="payment-element"></div>
<script src="https://js.stripe.com/v3/"></script>
<script>
const stripe = Stripe('pk_test_...');
const elements = stripe.elements({ clientSecret });
const paymentEl = elements.create('payment');
paymentEl.mount('#payment-element');
// Stripe injects a sandboxed iframe with allow="payment"
</script>

Payment SDKs inject their own sandboxed iframes. This is why allow="payment" exists.

36. Object element for SVG

<object data="diagram.svg" type="image/svg+xml"
        width="400" height="300"
        aria-label="Architecture diagram">
  <img src="diagram.png" alt="Architecture diagram (fallback)">
</object>

object with type="image/svg+xml" embeds an SVG file. img inside is the fallback.

37. Printing iframe content

function printFrame() {
  const frame = document.getElementById('report-frame');
  frame.contentWindow.print();
}
// For cross-origin iframes, this won't work — need server-side solution

Same-origin iframes can be printed via contentWindow.print(). Cross-origin iframes cannot.

38. iframe performance impact

<!-- Bad: loads immediately, blocks page -->
<iframe src="heavy-app.html"></iframe>

<!-- Good: lazy load -->
<iframe src="heavy-app.html" loading="lazy"></iframe>

<!-- Best: load on user interaction -->
<button onclick="loadWidget()">Load widget</button>
<div id="widget-slot"></div>
<script>
function loadWidget() {
  document.getElementById('widget-slot').innerHTML =
    '<iframe src="widget.html" title="Widget"></iframe>';
}
</script>

Each iframe is a full browsing context with its own memory and parsing. Defer loading whenever possible.

ADVANCED (39–50)

39. Federated identity (Login with X)

<!-- The browser handles the iframe for FedCM -->
const credential = await navigator.credentials.get({
  federated: {
    providers: [{ url: 'https://accounts.google.com', clientId: '...' }]
  }
});
// No iframe to manage directly — FedCM replaces the popup/iframe pattern

FedCM (Federated Credential Management) replaces iframes for identity login flows.

40. iframe Permissions Policy

<iframe src="app.html"
        allow="camera 'none'; microphone 'none'; geolocation 'none';
               fullscreen 'self'; payment https://stripe.com"
        title="Application"></iframe>

Permissions Policy (formerly Feature Policy) grants or denies access to browser features per-origin.

41. BroadcastChannel for iframe communication

// Works between same-origin iframes on the same origin
const channel = new BroadcastChannel('app-channel');
channel.postMessage({ type: 'theme-change', value: 'dark' });
channel.onmessage = e => console.log(e.data);

BroadcastChannel communicates between same-origin tabs, iframes, and workers without postMessage chains.

42. SharedWorker across iframes

// In each same-origin iframe:
const worker = new SharedWorker('shared.js');
worker.port.start();
worker.port.postMessage({ type: 'register' });
worker.port.onmessage = e => console.log(e.data);

SharedWorker instances are shared across same-origin pages/iframes — useful for shared state.

43. iframe storage isolation

<!-- Cross-origin iframes are storage-partitioned in modern browsers -->
<!-- Third-party cookies blocked by default in Chrome 120+ -->
<!-- Use Storage Access API to request first-party storage access -->
<script>
// Inside the iframe:
document.requestStorageAccess().then(() => {
  // Now can access cookies and localStorage as if first-party
});
</script>

Storage partitioning prevents cross-site tracking. Use Storage Access API for legitimate cross-origin storage needs.

44. Portals API (experimental)

<portal id="portal" src="next-page.html"></portal>
<script>
const portal = document.getElementById('portal');
portal.addEventListener('click', async () => {
  await portal.activate(); // instant page transition
});
</script>

Portals enable instant page transitions by pre-rendering the next page (experimental, Chrome only).

45. Fenced frames (Privacy Sandbox)

<!-- Experimental: for ads without cross-site tracking -->
<fencedframe src="ad.html" mode="opaque-ads"></fencedframe>

Fenced frames embed content without the embedded page knowing the embedding context. Part of Privacy Sandbox.

46. Service Worker intercepting iframe requests

// SW intercepts all fetch requests including those from iframes
self.addEventListener('fetch', e => {
  if (e.request.mode === 'navigate' && new URL(e.request.url).origin === location.origin) {
    e.respondWith(caches.match(e.request).then(r => r || fetch(e.request)));
  }
});

Service Workers control all same-origin fetches, including iframe navigation requests.

47. iframe in Shadow DOM

class EmbeddedWidget extends HTMLElement {
  connectedCallback() {
    const shadow = this.attachShadow({ mode: 'closed' });
    const frame = document.createElement('iframe');
    frame.src = this.getAttribute('src');
    frame.title = this.getAttribute('label') || 'Widget';
    frame.setAttribute('sandbox', 'allow-scripts');
    shadow.appendChild(frame);
  }
}
customElements.define('embedded-widget', EmbeddedWidget);

Encapsulate iframes inside custom elements with Shadow DOM for clean component APIs.

48. Clickjacking protection in practice

<!-- Attacker page (what you must prevent): -->
<style>iframe { opacity: 0; position: absolute; top: 0; left: 0; }</style>
<button>Click here to win!</button>
<iframe src="https://bank.com/transfer"></iframe>

<!-- Defense (on bank.com): -->
Content-Security-Policy: frame-ancestors 'none'
X-Frame-Options: DENY

Clickjacking overlays a transparent iframe over a decoy button. Prevent with CSP frame-ancestors.

49. Testing iframes in Cypress

// cypress/support/commands.js
Cypress.Commands.add('getIframeBody', (iframeSelector) => {
  return cy.get(iframeSelector)
    .its('0.contentDocument.body')
    .should('not.be.empty')
    .then(cy.wrap);
});

// In test:
cy.getIframeBody('#my-frame').find('button').click();

Cypress can interact with same-origin iframes by accessing their contentDocument directly.

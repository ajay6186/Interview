Use this decision tree to choose between article, section, and div.

8. Heading hierarchy in nested structures

13. When NOT to use article or section

<!-- DON'T use section as a generic container -->
<section class="wrapper">  <!-- Wrong: no heading, just layout -->

<!-- DO use div for layout/styling -->
<div class="wrapper">      <!-- Correct -->

<!-- DON'T use article for non-self-contained content -->
<article class="sidebar-widget"> <!-- Wrong if it's just related links -->

<!-- DO use aside for tangential content -->
<aside>Related links</aside>  <!-- Correct -->

Misusing section/article doesn't break the page but creates confusing landmark structure for AT users.

40. Intersection Observer for article view tracking

<script>
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        const title = entry.target.querySelector("h2, h3")?.textContent;
        analytics.track("article_viewed", { title });
        observer.unobserve(entry.target);
      }
    });
  }, { threshold: 0.5 });

  document.querySelectorAll("article").forEach(a => observer.observe(a));
</script>

IntersectionObserver on articles enables accurate content view tracking — fires when 50% of the article is visible.

42. Article reading progress indicator

<article id="main-article">
  ...long content...
</article>

<div id="reading-progress" role="progressbar" aria-valuenow="0"
     aria-valuemin="0" aria-valuemax="100" aria-label="Reading progress"
     style="position:fixed;top:0;left:0;height:3px;background:#0066cc;width:0">
</div>

<script>
  const article = document.getElementById("main-article");
  const bar = document.getElementById("reading-progress");
  window.addEventListener("scroll", () => {
    const rect = article.getBoundingClientRect();
    const progress = Math.min(100, Math.max(0,
      ((window.scrollY - article.offsetTop) / (article.offsetHeight - window.innerHeight)) * 100
    ));
    bar.style.width = progress + "%";
    bar.setAttribute("aria-valuenow", Math.round(progress));
  });
</script>

Reading progress bar tied to the article element. Uses role="progressbar" with aria-valuenow for accessibility.

  const btn = document.querySelector("[aria-controls='advanced-section']");
  btn.addEventListener("click", () => {
    const expanded = btn.getAttribute("aria-expanded") === "true";
    btn.setAttribute("aria-expanded", String(!expanded));
    document.getElementById("advanced-section").hidden = expanded;
  });
</script>

Collapsible sections: aria-expanded on button, aria-controls linking to section id. Toggle hidden attribute on the section.

48. Data-article attribute for JS targeting

49. Section with role="feed"

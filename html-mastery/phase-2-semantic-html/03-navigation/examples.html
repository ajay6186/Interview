BASIC (1–13)

1. Basic nav with ul/li/a

  • About
  • Blog
  • Contact

Nav + ul/li/a is the correct semantic structure. Screen readers announce it as a navigation landmark with a list.

2. Simple breadcrumb

  1. Home
  2. /
  3. Products
  4. /
  5. Widget Pro

Breadcrumb uses ol (ordered), aria-label="Breadcrumb", and aria-current="page" on the last item.

3. Pagination navigation

Prev 1 2 3 Next

Pagination nav: aria-label="Pagination", aria-current="page" on current page, rel="next"/"prev" on first/last links.

4. Active link with aria-current

  • Home
  • About
  • Blog

aria-current="page" announces "current" to screen readers and provides a CSS hook for styling the active link.

5. Footer navigation

  • Privacy Policy
  • Terms of Service
  • Sitemap

Footer nav needs its own aria-label to distinguish it from main nav in the landmarks list.

6. Skip navigation link

<nav>...</nav>

Skip links jump over repeated navigation for keyboard users. Must be the first focusable element. Required by WCAG 2.4.1.

7. Table of contents nav

Table of contents uses nested ol inside nav. Hierarchy mirrors the document heading structure.

8. Language switcher nav

9. rel="prev" and rel="next" on pagination

<nav aria-label="Article navigation">
  <a href="/articles/previous" rel="prev">← Previous Article</a>
  <a href="/articles/next" rel="next">Next Article →</a>
</nav>

rel="prev" and rel="next" inform search engines about the relationship between pages in a series.

10. aria-current values beyond "page"

<!-- Current page in site nav -->
<a aria-current="page">Dashboard</a>

<!-- Current step in a wizard -->
<a aria-current="step">Step 2: Payment</a>

<!-- Currently selected item (non-page) -->
<a aria-current="true">Alice Johnson</a>

<!-- Current date in calendar -->
<td aria-current="date">24</td>

aria-current accepts: page, step, location, date, time, or true/false. Choose the most specific value.

31. Active section tracking with IntersectionObserver

<nav aria-label="On this page" id="toc-nav">
  <ul>
    <li><a href="#intro" id="nav-link-intro">Introduction</a></li>
    <li><a href="#usage" id="nav-link-usage">Usage</a></li>
  </ul>
</nav>
<script>
  const observer = new IntersectionObserver(entries => {
    entries.forEach(e => {
      const link = document.getElementById("nav-link-" + e.target.id);
      if (link) link.setAttribute("aria-current", e.isIntersecting ? "true" : "false");
      if (!e.isIntersecting) link?.removeAttribute("aria-current");
    });
  }, { threshold: 0.5 });
  document.querySelectorAll("section[id]").forEach(s => observer.observe(s));
</script>

Active section detection via IntersectionObserver. Sets aria-current="true" on the nav link when its section is 50% visible.

34. Focus trap for mobile nav overlay

<script>
  function trapFocus(element) {
    const focusable = element.querySelectorAll(
      'a[href], button:not([disabled]), input, [tabindex]:not([tabindex="-1"])'
    );
    const first = focusable[0];
    const last = focusable[focusable.length - 1];

    element.addEventListener("keydown", e => {
      if (e.key === "Escape") closeNav();
      if (e.key !== "Tab") return;
      if (e.shiftKey && document.activeElement === first) {
        e.preventDefault(); last.focus();
      } else if (!e.shiftKey && document.activeElement === last) {
        e.preventDefault(); first.focus();
      }
    });
  }
</script>

Focus traps prevent keyboard users from leaving an open overlay nav. Escape closes; Tab cycles only within the nav.

35. Navigation with prefetch on hover

<nav aria-label="Main">
  <ul>
    <li><a href="/products">Products</a></li>
    <li><a href="/analytics">Analytics</a></li>
  </ul>
</nav>
<script>
  document.querySelectorAll("nav a[href]").forEach(link => {
    link.addEventListener("mouseenter", () => {
      if (document.querySelector(`link[href="${link.href}"]`)) return;
      const prefetch = document.createElement("link");
      prefetch.rel = "prefetch";
      prefetch.href = link.href;
      document.head.appendChild(prefetch);
    }, { once: true });
  });
</script>

Hover-prefetch pre-loads nav destinations during user intent, making navigation feel instant. Fires once per link.

36. SPA navigation with focus management

<nav aria-label="App navigation">

    <li><a href="/settings" data-route>Settings</a></li>

<script>
  document.querySelectorAll("[data-route]").forEach(link => {
    link.addEventListener("click", e => {
      e.preventDefault();
      navigateTo(link.href);
      // After render:

</script>

37. Navigation audit script

<script>
  // Audit all navigation landmarks
  const navs = document.querySelectorAll("nav, [role='navigation']");
  console.log(`Found ${navs.length} navigation landmark(s):`);
  navs.forEach((nav, i) => {

ADVANCED (39–50)

39. Navigation in web component

<site-navigation current-page="/dashboard"></site-navigation>

<script>
  class SiteNavigation extends HTMLElement {
    static observedAttributes = ["current-page"];
    connectedCallback() {
      this.attachShadow({ mode: "open" });
      this.render();
    }
    attributeChangedCallback() { this.render(); }
    render() {
      const current = this.getAttribute("current-page") || "/";
      this.shadowRoot.innerHTML = `
        <nav aria-label="Main navigation">
          <ul>
            ${["/dashboard", "/settings", "/help"].map(path => `
              <li><a href="${path}" ${path === current ? 'aria-current="page"' : ''}>
                ${path.slice(1)}
              </a></li>
            `).join("")}
          </ul>
        </nav>
      `;
    }
  }
  customElements.define("site-navigation", SiteNavigation);
</script>

Web component navigation with aria-current set from an attribute. Shadow DOM nav is still traversable by AT.

43. Visible focus styles for nav links

<style>
  nav a { color: inherit; text-decoration: none; }
  nav a:hover { text-decoration: underline; }
  /* Use :focus-visible for keyboard focus only */
  nav a:focus-visible {
    outline: 3px solid #0066cc;
    outline-offset: 2px;
    border-radius: 2px;
  }
  /* Remove outline on mouse click */
  nav a:focus:not(:focus-visible) {
    outline: none;
  }
</style>

:focus-visible shows focus ring only for keyboard navigation, not mouse clicks. The outline must be visible against all background colors (WCAG 2.4.7).

44. Persistent active state with localStorage

<nav aria-label="Main" id="persist-nav">
  <ul>
    <li><a href="/products">Products</a></li>
    <li><a href="/analytics">Analytics</a></li>
  </ul>
</nav>
<script>
  const nav = document.getElementById("persist-nav");
  nav.addEventListener("click", e => {
    if (e.target.tagName === "A") {
      localStorage.setItem("lastNav", e.target.href);
    }
  });
  const last = localStorage.getItem("lastNav");
  if (last) {
    const link = nav.querySelector(`a[href="${last}"]`);
    link?.setAttribute("aria-current", "page");
  }
</script>

Persist the last visited nav section across page loads. Useful for tabbed applications where the active section is remembered.

49. Navigation with roles audit

<script>
  // Full navigation accessibility audit
  const issues = [];
  document.querySelectorAll("nav").forEach(nav => {
    if (!nav.getAttribute("aria-label") && !nav.getAttribute("aria-labelledby")) {
      issues.push(`Nav without label: ${nav.outerHTML.slice(0, 50)}...`);
    }
  });
  const navs = document.querySelectorAll("nav");
  const labels = new Set([...navs].map(n => n.getAttribute("aria-label")));
  if (labels.size !== navs.length) {
    issues.push("Duplicate aria-label values on nav elements");
  }
  if (issues.length) {
    console.warn("Navigation issues:", issues);
  } else {
    console.log("All navigation landmarks are correctly labeled.");
  }
</script>

Automated nav audit: checks for missing labels and duplicate label values across all nav elements.

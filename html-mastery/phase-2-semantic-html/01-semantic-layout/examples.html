Examples 2.1 — Semantic Layout

BASIC (1–13)

1. header element

My Website

Your tagline here

<header> contains introductory content or navigation aids. Implicit ARIA role: banner (when a direct child of body).

2. nav element

Home About Blog Contact

<nav> wraps major navigation blocks. Implicit ARIA role: navigation. Add aria-label when multiple navs exist.

3. main element

<main>
  <h2>Article Title</h2>
  <p>The primary content of the page lives here.</p>
</main>

<main> identifies the dominant content. Only ONE per page. Implicit ARIA role: main. Skip links target this element.

4. footer element

© 2026 Company Name. Privacy Policy · Terms

<footer> contains information about its parent section: author, copyright, links. Implicit ARIA role: contentinfo (when direct child of body).

5. aside element

<aside aria-label="Related links">
  <h2>Related Articles</h2>
  <ul>
    <li><a href="#">Article One</a></li>
    <li><a href="#">Article Two</a></li>
  </ul>
</aside>

<aside> is tangentially related to main content. Implicit ARIA role: complementary. Use for sidebars, related links, ads.

6. section element

<section>
  <h2>Services</h2>
  <p>We offer design, development, and consulting.</p>
</section>

<section> is a thematic grouping of content with a heading. Don't use it as a generic container — use <div> for that.

7. article element

<article>
  <h2>Post Title</h2>
  <p>Published: <time datetime="2026-03-24">March 24, 2026</time></p>
  <p>Content of the article...</p>
</article>

<article> is self-contained content that could be syndicated independently (blog posts, news articles, comments, widgets).

8. div vs section vs article

<div>               <!-- No semantic meaning; use for styling/JS -->
<section>          <!-- Thematic group; needs a heading -->
<article>          <!-- Self-contained, syndicate-able content -->

Rule of thumb: Would it make sense in an RSS feed? → article. Does it have a thematic heading? → section. Otherwise → div.

9. The skip navigation link

<a href="#main-content" class="skip-link">Skip to main content</a>
<header>...</header>
<main id="main-content">...</main>

Skip links allow keyboard users to bypass navigation. Position absolutely and reveal on :focus. Required for WCAG 2.4.1.

10. header inside article (nested header)

<article>
  <header>
    <h2>Article Title</h2>
    <p>By <cite>Author Name</cite> · <time datetime="2026-01-01">Jan 1, 2026</time></p>
  </header>
  <p>Article body...</p>
  <footer><p>Tags: HTML, Semantics</p></footer>
</article>

<header> and <footer> can appear inside <article> and <section>. They do NOT carry landmark roles in these contexts.

11. Multiple nav elements with labels

<nav aria-label="Main navigation">
  <!-- primary site links -->
</nav>

<nav aria-label="Breadcrumb">
  <!-- breadcrumb trail -->
</nav>

<nav aria-label="Pagination">
  <!-- page 1, 2, 3... -->
</nav>

Multiple <nav> elements need distinct aria-label values so screen reader users can differentiate them in the landmarks list.

12. Landmark roles implicit in semantic elements

<header>      → role="banner"       (when direct child of body)
<nav>         → role="navigation"
<main>        → role="main"
<aside>       → role="complementary"
<footer>      → role="contentinfo"   (when direct child of body)
<form>        → role="form"          (only if it has an accessible name)
<section>     → role="region"        (only if it has an accessible name)

Semantic HTML5 elements carry implicit ARIA landmark roles. Explicit role= attributes are redundant but acceptable for older AT.

13. section needs accessible name to become a landmark

<!-- NOT a landmark (no name) -->
<section>...</section>

<!-- IS a landmark (has aria-labelledby) -->
<section aria-labelledby="services-heading">
  <h2 id="services-heading">Our Services</h2>
  ...
</section>

A <section> only becomes a region landmark when it has an accessible name via aria-labelledby or aria-label.

INTERMEDIATE (14–26)

14. Two-column layout (main + aside)

Main Content

Primary article content goes here.

Sidebar

Related links.

Classic two-column layout: flex main + aside. The visual and semantic roles align correctly.

15. Holy grail layout

<body>
  <header>...</header>
  <nav>...</nav>
  <div class="content-area">
    <aside class="left-sidebar">...</aside>
    <main>...</main>
    <aside class="right-sidebar">...</aside>
  </div>
  <footer>...</footer>
</body>

The "holy grail" layout: header, 3-column middle (left sidebar + main + right sidebar), footer. Two asides must have distinct aria-labels.

16. header as article header (not page banner)

<article>
  <header>
    <!-- This header's role is NOT "banner" — it belongs to the article -->
    <h2>Article Title</h2>
    <address>By <a href="/author">Jane Doe</a></address>
  </header>
  ...
</article>

When <header> is inside <article>, <section>, <nav>, or <aside>, it does NOT have the banner role.

17. Sticky header pattern

<header style="position: sticky; top: 0; z-index: 100; background: white;">
  <h1>Site Name</h1>
  <nav>...</nav>
</header>

Sticky headers stay visible during scroll. Ensure they don't overlap focused elements — add scroll-margin-top to sections.

18. aria-label on section to create a named region

<section aria-label="Customer testimonials">
  <h2>What Our Customers Say</h2>
  ...
</section>

<!-- Or with aria-labelledby -->
<section aria-labelledby="testimonials-h">
  <h2 id="testimonials-h">What Our Customers Say</h2>
  ...
</section>

aria-labelledby preferred over aria-label when there's a visible heading — it ties the heading text to the landmark name.

19. nav inside header vs standalone

<!-- Nav inside header (common for site-wide nav) -->
<header>
  <h1>Site Name</h1>
  <nav aria-label="Main">...</nav>
</header>

<!-- Nav outside header (valid, standalone navigation) -->
<nav aria-label="Main">...</nav>

Both patterns are valid. Nav inside header is more common; nav outside is correct for dedicated navigation bars.

20. main with id for skip link target

<a href="#main" class="skip-link">Skip to main content</a>

<header>...</header>
<nav>...</nav>
<main id="main" tabindex="-1">
  <!-- tabindex="-1" allows programmatic focus, needed for some screen readers -->
  ...
</main>

tabindex="-1" on <main> allows it to receive focus from the skip link in browsers that don't focus non-focusable elements on hash change.

21. footer in article for metadata

<article>
  <h2>Post Title</h2>
  <p>Content...</p>
  <footer>
    <p>
      Published <time datetime="2026-03-24">March 24, 2026</time> ·
      Tags: <a href="#">HTML</a>, <a href="#">Semantics</a>
    </p>
  </footer>
</article>

<footer> inside <article> contains meta info about the article. It doesn't carry the contentinfo landmark role here.

22. Using address inside footer

<footer>
  <address>
    <a href="mailto:contact@example.com">contact@example.com</a><br>
    123 Main St, Springfield, IL
  </address>
  <p><small>© 2026 Company Name</small></p>
</footer>

<address> provides contact info for the nearest <article> or <body>. Inside the page footer, it represents site-wide contact info.

23. Aside for pull quotes

Performance is a critical aspect of web development. As Paul Lewis noted:

"The fastest code is the code that doesn't run."

Every unnecessary script adds to the execution budget.

<aside> inside <article> can mark pull quotes or supplementary callouts tangential to the main article flow.

24. Landmark list for screen reader navigation

<!-- What AT users see in their landmarks menu when page is structured correctly: -->
<!--
  banner (header)
  navigation — Main navigation
  navigation — Breadcrumb
  main
  complementary — Related articles
  contentinfo (footer)
-->

Well-structured pages provide meaningful landmarks that AT users can navigate via shortcut keys (F6, or SR-specific commands).

25. Reducing div soup with semantic elements

<!-- Before: div soup -->
<div class="header">
  <div class="nav">...</div>
</div>
<div class="main-content">
  <div class="sidebar">...</div>
  <div class="content">...</div>
</div>
<div class="footer">...</div>

<!-- After: semantic HTML -->
<header>
  <nav>...</nav>
</header>
<div class="layout">
  <aside>...</aside>
  <main>...</main>
</div>
<footer>...</footer>

Replacing divs with semantic elements provides AT landmarks, improves SEO, and makes code self-documenting.

ADVANCED (39–50)

39. Detecting landmarks with JavaScript

<script>
  // Find all landmark elements
  const landmarks = document.querySelectorAll(
    "header, nav, main, aside, footer, [role='banner'], [role='navigation'], " +
    "[role='main'], [role='complementary'], [role='contentinfo'], [role='search']"
  );
  console.log("Landmarks found:", landmarks.length);
  landmarks.forEach(el => {
    console.log(el.tagName, el.getAttribute("aria-label") || "(no label)");
  });
</script>

You can programmatically audit landmark structure. Real tools: axe DevTools, Landmark Navigation browser extension.

40. Scroll-to-section with smooth scroll

<nav aria-label="Page sections">
  <a href="#intro">Introduction</a>
  <a href="#features">Features</a>
  <a href="#pricing">Pricing</a>
</nav>

<main>
  <section id="intro" aria-labelledby="intro-h">
    <h2 id="intro-h">Introduction</h2>...
  </section>
  <section id="features" aria-labelledby="features-h"
           style="scroll-margin-top: 80px;">
    <h2 id="features-h">Features</h2>...
  </section>
</main>

<style>html { scroll-behavior: smooth; }</style>

scroll-margin-top offsets anchor scrolling for sticky headers. scroll-behavior: smooth is a CSS property that enables smooth navigation.

41. Inert attribute for background content

<main inert aria-hidden="true">
  <!-- Page content made non-interactive while modal is open -->
</main>
<dialog open>
  <!-- Modal content -->
  <button>Close</button>
</dialog>

The inert attribute (HTML spec) removes all interactivity from a subtree: no focus, no clicks, no AT traversal. Ideal for modal dialogs.

42. Document outline algorithm

<!-- The proposed document outline (not universally implemented by AT):
  1. Introduction        ← <h1> in body
  2. Getting Started     ← <h2>
     2.1 Installation    ← <h3>
     2.2 Configuration   ← <h3>
  3. API Reference       ← <h2>
-->
<!-- Each <section> was intended to restart the outline, but AT support never arrived. -->
<!-- SAFE: use sequential h1 h2 h3... regardless of sectioning elements. -->

The HTML5 document outline algorithm (each section restarts with h1) was never implemented by AT. Use sequential heading levels for safety.

43. BEM class naming alongside semantic elements

<header class="site-header">
  <div class="site-header__logo">...</div>
  <nav class="site-header__nav" aria-label="Main">...</nav>
</header>
<main class="page-main page-main--wide">...</main>
<aside class="sidebar sidebar--sticky">...</aside>

BEM (Block__Element--Modifier) classes work alongside semantic elements — classes are for styling, elements are for semantics.

44. View Transitions API with semantic elements

<style>
  main { view-transition-name: main-content; }
</style>
<script>
  document.addEventListener("click", e => {
    const link = e.target.closest("a[href]");
    if (!link || !document.startViewTransition) return;
    e.preventDefault();
    document.startViewTransition(() => {
      navigation.navigate(link.href);
    });
  });
</script>

The View Transitions API animates between page states. Assigning view-transition-name to <main> creates a smooth content transition.

45. Landmark order and visual order alignment

<!-- CORRECT: DOM order matches visual order -->
<header>Header</header>
<main>Main content</main>
<aside>Sidebar</aside>
<footer>Footer</footer>

<!-- CSS places sidebar visually to the left, but DOM order = reading order -->
<style>
  body { display: grid; grid-template: "h h" "m s" "f f"; }
  header { grid-area: h; } main { grid-area: m; }
  aside { grid-area: s; } footer { grid-area: f; }
</style>

Use CSS grid/flex to reorder visually while keeping DOM order logical — assistive technologies and keyboard navigation follow DOM order.

47. ARIA landmark shortcut key reference

<!-- Screen reader keyboard shortcuts for landmarks:
  JAWS:      R / Shift+R  — next/prev region
  NVDA:      D / Shift+D  — next/prev landmark
  VoiceOver: VO+U opens rotor, select Landmarks
  Narrator:  Caps+F5      — landmarks list
-->

<!-- Users can also use the Landmark Navigation browser extension -->

AT users rely on these shortcuts to quickly scan page structure. Well-named landmarks dramatically improve their experience.

48. hidden vs aria-hidden on layout elements

<aside hidden>
  <!-- Completely removed from all users (visual + AT) -->
</aside>

<aside aria-hidden="true">
  <!-- Visible to sighted users but hidden from AT -->
  <!-- Use for decorative panels, animated backgrounds -->
</aside>

<aside class="visually-hidden">
  <!-- Hidden from sighted users but announced by AT -->
  <!-- Use for supplementary context needed only by AT -->
</aside>

Three hiding techniques with different visibility tradeoffs. Choose based on who needs the content.

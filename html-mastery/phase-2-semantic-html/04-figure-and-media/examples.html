Examples 2.4 — Figure & Media

BASIC (1–13)

1. figure with figcaption

<figure> wraps self-contained content with an optional <figcaption>. The caption supplements (not duplicates) the alt text.

2. video with controls

<video controls width="640" height="360">
  <source src="intro.mp4" type="video/mp4">
  <source src="intro.webm" type="video/webm">
  <p>Your browser does not support video. <a href="intro.mp4">Download it</a>.</p>
</video>

Multiple <source> elements provide format fallbacks. The paragraph is shown when video is not supported at all.

3. video with poster

<video controls poster="thumbnail.jpg" width="640" height="360">
  <source src="video.mp4" type="video/mp4">
</video>

poster shows an image before the user plays the video. Use a relevant, enticing frame from the video.

4. audio with controls

<audio controls>
  <source src="podcast.mp3" type="audio/mpeg">
  <source src="podcast.ogg" type="audio/ogg">
  <p>Your browser does not support audio.</p>
</audio>

Always provide controls for audio. Never use autoplay without muted — it's disruptive and fails WCAG 1.4.2.

5. figcaption placement (below or above)

<!-- Caption below (most common) -->
<figure>
  <img src="chart.png" alt="Sales chart">
  <figcaption>Figure 1: Annual sales data.</figcaption>
</figure>

<!-- Caption above -->
<figure>
  <figcaption>Table 1: Employee data.</figcaption>
  <table>...</table>
</figure>

<figcaption> can be first or last child of <figure>. Below is conventional for images; above is common for tables.

6. figure for a quote (non-image)

<figure> is not just for images — use it for blockquotes, code listings, diagrams, or any self-contained content with a caption.

7. video with autoplay muted loop (background video)

<video autoplay muted loop playsinline
       aria-hidden="true"
       width="1200" height="675">
  <source src="background.mp4" type="video/mp4">
</video>

Background videos must be muted (browser requirement for autoplay), have aria-hidden="true" (decorative), and use playsinline for iOS.

8. video track for captions

<video controls width="640" height="360">
  <source src="lecture.mp4" type="video/mp4">
  <track kind="captions" src="captions-en.vtt" srclang="en" label="English" default>
  <track kind="captions" src="captions-fr.vtt" srclang="fr" label="Français">
  <track kind="descriptions" src="descriptions.vtt" srclang="en" label="Audio descriptions">
</video>

<track> provides captions/subtitles/descriptions. kind values: subtitles, captions, descriptions, chapters, metadata. Required for WCAG 1.2.2.

9. figure for a code listing

Code listings inside <figure> get a <figcaption> for reference (e.g., "Listing 3") that can be cross-referenced in prose.

10. audio with loop and preload

<audio controls loop preload="metadata">
  <source src="background-music.mp3" type="audio/mpeg">
</audio>

preload="metadata" loads only duration/dimensions without full download. preload="none" defers until user interaction.

11. figure without figcaption (self-contained image)

<figure>
  <img src="logo.svg" alt="Acme Co logo">
</figure>

<figcaption> is optional. Use <figure> without it when the content is self-contained but the caption would be redundant with the alt text.

12. video without controls (controlled by JS)

<video id="hero-video" muted playsinline aria-hidden="true">
  <source src="hero.mp4" type="video/mp4">
</video>
<button onclick="document.getElementById('hero-video').play()">Play</button>
<button onclick="document.getElementById('hero-video').pause()">Pause</button>

Custom video controls via JavaScript. Must provide accessible buttons — never remove native controls without replacing them.

13. figure with multiple images (gallery)

A <figure> can contain multiple images sharing a single caption. The caption covers the group as a whole.

INTERMEDIATE (14–26)

14. Responsive video with aspect-ratio

    <source src="video.mp4" type="video/mp4">
  </video>
  <figcaption>Demo: Building a responsive layout in 10 minutes.</figcaption>
</figure>

aspect-ratio: 16/9 on video ensures correct proportions at any width. Combine with width: 100% for responsive behaviour.

15. Picture element with multiple sources

<picture> inside <figure> for art direction. The <img> is the fallback and carries the alt text.

16. Video with chapter tracks

<video controls width="640" height="360">
  <source src="tutorial.mp4" type="video/mp4">
  <track kind="chapters" src="chapters.vtt" srclang="en" label="Chapters" default>
  <track kind="captions" src="captions.vtt" srclang="en" label="English CC" default>
</video>

<!-- chapters.vtt:
WEBVTT

00:00:00.000 --> 00:05:30.000
Introduction

00:05:30.000 --> 00:15:00.000
Setting Up
-->

kind="chapters" adds chapter markers to the video timeline for navigation. Requires a WebVTT file.

17. Video with crossorigin for CORS

<video controls crossorigin="anonymous" width="640" height="360">
  <source src="https://cdn.example.com/video.mp4" type="video/mp4">
  <track kind="captions" src="https://cdn.example.com/captions.vtt"
         srclang="en" label="English" default>
</video>

crossorigin="anonymous" enables loading VTT files from a different origin. Required when captions are on a CDN with CORS headers.

24. Lazy-loaded figure images

<figure>
  <img src="large-photo.jpg"
       alt="Aerial view of the city at night"
       loading="lazy"
       width="1200" height="800">
  <figcaption>Photo: City skyline, taken from 500m altitude.</figcaption>

loading="lazy" on figure images defers loading until near the viewport. Never use on above-the-fold hero images.

25. Video with MediaSource API placeholder

<video id="adaptive-video" controls width="640" height="360">
  <!-- Src set dynamically via MediaSource Extensions (MSE) -->
  <p>Your browser doesn't support adaptive streaming.</p>
</video>
<script>
  // HLS/DASH players (hls.js, dash.js) use MSE to set src dynamically
  if (typeof Hls !== 'undefined' && Hls.isSupported()) {
    const hls = new Hls();
    hls.loadSource("stream.m3u8");
    hls.attachMedia(document.getElementById("adaptive-video"));
  }
</script>

Adaptive bitrate streaming (HLS/DASH) requires JavaScript libraries. The HTML video element is still the host element.

39. MediaSession API for audio

<audio id="media-audio" controls src="podcast.mp3"></audio>
<script>
  if ('mediaSession' in navigator) {
    navigator.mediaSession.metadata = new MediaMetadata({
      title: 'Episode 42: HTML Semantics',
      artist: 'Web Dev Podcast',
      album: 'Season 3',
      artwork: [{ src: 'cover.jpg', sizes: '512x512', type: 'image/jpeg' }]
    });
    navigator.mediaSession.setActionHandler('play', () => document.getElementById('media-audio').play());
    navigator.mediaSession.setActionHandler('pause', () => document.getElementById('media-audio').pause());
  }
</script>

MediaSession API sets metadata for OS-level media controls (lock screen, notifications). Works with both audio and video elements.

40. Video Picture-in-Picture API

<video id="pip-video" controls src="lecture.mp4" width="640" height="360"></video>
<button id="pip-btn">Picture-in-Picture</button>

<script>
  const video = document.getElementById("pip-video");
  document.getElementById("pip-btn").addEventListener("click", async () => {
    if (document.pictureInPictureElement) {
      await document.exitPictureInPicture();
    } else if (document.pictureInPictureEnabled) {
      await video.requestPictureInPicture();
    }
  });
  video.addEventListener("enterpictureinpicture", () => {
    document.getElementById("pip-btn").textContent = "Exit Picture-in-Picture";

</script>

Picture-in-Picture lets users watch video in a floating overlay while browsing other content. Fully accessible via the native window.

44. Intersection Observer for lazy video loading

<video data-src="heavy-video.mp4" controls width="640" height="360" poster="poster.jpg">
  <source data-src="heavy-video.mp4" type="video/mp4">
</video>

<script>
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (!entry.isIntersecting) return;
      const video = entry.target;
      video.querySelector("source").src = video.querySelector("source").dataset.src;
      video.load();
      observer.unobserve(video);
    });
  }, { threshold: 0.1 });
  document.querySelectorAll("video[data-src]").forEach(v => observer.observe(v));
</script>

Lazy-load video source via IntersectionObserver. Use data-src to defer setting the src until the element is near the viewport.

47. AudioWorklet for real-time processing

<audio id="processed-audio" controls></audio>
<button id="process-btn">Apply Effect</button>
<script>
  document.getElementById("process-btn").addEventListener("click", async () => {
    const ctx = new AudioContext();
    await ctx.audioWorklet.addModule("processor.js");
    const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
    const source = ctx.createMediaStreamSource(stream);
    const processor = new AudioWorkletNode(ctx, "my-processor");
    source.connect(processor).connect(ctx.destination);
  });
</script>

AudioWorklet processes audio on a dedicated thread. Much more powerful than ScriptProcessorNode (deprecated). Advanced API for audio effects.

48. Video analytics with events

<video id="tracked-video" controls src="product-demo.mp4"></video>
<script>
  const video = document.getElementById("tracked-video");
  const events = ["play", "pause", "ended", "seeking", "error"];
  events.forEach(type => {
    video.addEventListener(type, () => {
      analytics.track("video_" + type, {
        src: video.src,
        currentTime: video.currentTime,
        duration: video.duration,
        playbackRate: video.playbackRate,
        volume: video.volume,
        muted: video.muted
      });
    });
  });
</script>

Track video engagement with native events: play, pause, ended, seeking, error. currentTime gives exact position for drop-off analysis.

49. Automated captions with WebSpeech API

<video id="live-video" controls width="640" height="360"></video>
<div id="live-captions" role="status" aria-live="assertive" aria-atomic="false"
     style="min-height:40px;background:#000;color:#fff;padding:8px;font-size:18px">
</div>
<script>
  const recognition = new (window.SpeechRecognition || window.webkitSpeechRecognition)();
  recognition.continuous = true;
  recognition.interimResults = true;
  recognition.onresult = e => {
    const transcript = Array.from(e.results).map(r => r[0].transcript).join(" ");
    document.getElementById("live-captions").textContent = transcript;
  };
  recognition.start();
</script>

Live captions via Web Speech API. The aria-live="assertive" region announces new caption text immediately to screen readers.
